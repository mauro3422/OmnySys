# v0.9.385 - Sprint 16: MCP Discovery Resources and Reconnect Auto-Heal

## Changed

- Added read-only MCP resources for discovery so clients can inspect live status, health, session state, tool inventory, and schema before calling a tool.
- Hardened Streamable HTTP recovery with shared event-store resumability and stronger session deduplication so reconnect loops can be repaired automatically.
- Documented the Codex connector failure mode where repeated `Reconnecting...` retries can coexist with a later successful tool call, because the successful call does not prove the earlier loop was harmless.
