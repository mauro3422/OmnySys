# v0.9.182 - Sprint 16: Tree-Sitter Freshness and DB-Only File Surfaces

## What changed
- Keyed the Tree-Sitter integration cache by file content hash so same-path edits no longer reuse stale analysis.
- Aggregated `scopeType` from all Tree-Sitter shared-state accesses instead of taking the first hit.
- Made single-file file analysis DB-only by default and gated `system_files` enrichment behind the canonical system-map coverage check.
- Populated `files.module_name` from the canonical module-name derivation instead of persisting `null`.
- Hardened the export path so legacy JSON is opt-in only and SQLite remains the default source of truth.

## Why
- Tree-Sitter metadata was being reused too aggressively when a file changed in place.
- File-level analysis could still trust a mirrored surface even when the canonical coverage was not healthy.
- The schema already promised module identity and freshness signals, so the pipeline should project them explicitly instead of leaving them empty.

## Validation
- `node --check` passed for the touched pipeline, query, and metadata files.
- `vitest` passed for the pipeline-group regression covering scope aggregation and module-name derivation.
- `get_recent_errors()` stayed clean after the edit.
