# v0.9.85 - FileWatcher Alerts, Runtime Guardrails, and MCP HTTP Hardening

**Date:** 2026-03-04  
**Type:** Stability + Technical Debt Reduction  
**Status:** Released

---

## Summary

This release strengthens the event-driven quality loop around `fileWatcher` without forcing heavy full-project recomputation.

Main goals completed:
- Persist watcher-driven alerts to SQLite for follow-up and backlog iteration.
- Add lightweight impact and duplication signals after each file change.
- Harden HTTP JSON parsing and avoid noisy stack dumps for malformed payloads.
- Keep MCP tool contracts updated and tested.

---

## What Changed

### 1) FileWatcher impact and duplicate risk signals

`src/core/file-watcher/handlers/file-handlers.js`

- Added `detectImpactWaveForFile(...)`:
  - Computes local change impact score/level (`low|medium|high`) after create/modify.
  - Uses changed atoms + related files + broken imports + broken callers.
  - Emits `impact:wave` event and warning logs.
- Added `detectDuplicateRiskForFile(...)`:
  - Detects likely duplicate symbols for the changed file.
  - Emits `duplicate:risk` event and warning logs.
- Added SQLite persistence helpers:
  - `persistWatcherIssue(...)`
  - `clearWatcherIssue(...)`
- Persisted watcher issue types:
  - `watcher_impact_wave`
  - `watcher_duplicate_risk`

### 2) Runtime watcher error persistence (new)

`src/core/file-watcher/lifecycle/change-processing.js`

- Added persistence for processing failures:
  - `issue_type: watcher_runtime_error`
  - `severity: high`
  - Message prefixed with `[watcher]`
- Automatic cleanup of `watcher_runtime_error` when processing later succeeds.

This closes the gap where watcher runtime errors could be visible only in transient logs.

### 3) FileWatcher wiring fix

`src/core/file-watcher/handlers.js`

- Re-exported missing methods:
  - `detectImpactWaveForFile`
  - `detectDuplicateRiskForFile`

Fixes runtime error:
- `this.detectImpactWaveForFile is not a function`

### 4) Orchestrator notification for impact wave

`src/core/orchestrator/lifecycle/watchers/file-watcher.js`

- Added listener for `impact:wave`.
- Broadcasts websocket event payload for real-time clients.
- Added listener for `duplicate:risk`.
- Added listener for `change:error`.
- Added listener for `import:orphaned`.
- Added listener for raw watcher `error`.
- These signals are now broadcast through websocket so watcher warnings/errors are visible to clients immediately.

### 5) MCP aggregate and atomic write guardrails

`src/layer-c-memory/mcp/tools/aggregate-metrics.js`

- Added `aggregationType: watcher_alerts`.
- Reads watcher issues from `semantic_issues` (`message LIKE '[watcher]%'`).

`src/layer-c-memory/mcp/tools/atomic-edit/atomic-writer-tool.js`

- Added `failOnDuplicate` option.
- Can hard-fail write on duplicate symbol risk (`DUPLICATE_SYMBOL_RISK`).

`src/layer-c-memory/mcp/tools/index.js`

- Updated tool descriptions for active/deprecated modes.
- Added `watcher_alerts` to aggregate options.
- Added `failOnDuplicate` to atomic_write schema.

### 6) MCP HTTP parse-error hardening

`src/layer-c-memory/mcp-http-server.js`

- Improved `conditionalJson` handling.
- Malformed JSON now returns clean JSON-RPC parse error (`-32700`) instead of noisy Express HTML stack output.
- Logs concise warning:
  - `[MCP JSON PARSE] POST /mcp: ...`

### 7) MCP registry test update

`tests/unit/layer-c/mcp/tools-registry.test.js`

- Updated expected known tool:
  - `mcp_omnysystem_get_atom_schema` -> `mcp_omnysystem_get_schema`
