# v0.9.377 - Sprint 16: Folderization Naming Split and Live-Row Drift Suppression

## Summary
- Split the folderization naming logic into a small public wrapper and a dedicated helper module so barrel detection, rename planning, and collision handling stay readable without tripping the file-size watcher.
- Tightened barrel-aware folderized-family selection so `index.js` candidates resolve to the highest-signal family in the directory instead of a low-signal fallback.
- Disabled the silent direct-write fallback for folderized import rewrites by default and made partial rewrite failures explicit.
- Hooked runtime reindexing into `syncRuntimeTableHealthIssues()` so freshly saved atoms/files also reconcile live support tables in the same cycle.
- Suppressed transient relation-only `runtime_table_health_live_row_drift` alerts while Phase 2 is still settling, so the watcher does not keep surfacing non-actionable drift during busy reindex windows.
- Hardened atomic editor emits and error-guardian recovery paths so missing `emit` callbacks and missing `projectPath` values no longer crash the recovery stack.
