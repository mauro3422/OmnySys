# v0.9.418

## Added
- Canonical promotion layer above folderization to evaluate folderized families and emergent surfaces for API promotion without mixing that logic into the folderization tool itself.
- Dedicated `mcp_omnysystem_get_canonical_promotion_report` tool to combine system inventory, folderization evidence, and promotion readiness into a reusable plan.
- Canonical promotion surfaced in metrics snapshots, health dashboards, status payloads, and the status table.
- System inventory surfaced alongside tool inventory and cache policy so the runtime can classify canonical surfaces, emergent systems, bridges, wrappers, and promotion candidates.

## Notes
- Folderization remains the structural mover and normalizer.
- Canonical promotion is the decision layer that decides what can be elevated into a canonical API.
- The new promotion plan is reusable by downstream consumers and can be cached by fingerprint.
