# v0.9.149 - Sprint 15: Canonical Atom-ID Reconciliation

**Date:** 2026-03-19

## Summary

This hotfix restores canonical alignment between `atoms.id` and `atom_relations` by preserving the atom's native ID format during relation persistence and migrating the current mixed-ID rows in SQLite back into a consistent joinable state.

## Key Changes

1.  **ID Preservation**: Relation writers now preserve the atom's existing ID format instead of forcing absolute or relative paths into a single shape.
2.  **Legacy Cleanup**: Relation cleanup now removes both legacy and current `source_id` variants before re-inserting calls, preventing mixed-ID drift from coming back.
3.  **Current DB Migration**: The live database's mixed call relations were normalized so `call_graph` and `atom_relations` are aligned again.
4.  **Runtime Health**: Database health returns to `A+` with zero orphan call relations and a fully aligned canonical call projection.

## Notes

- The bug was not data corruption. It was an ID-shape mismatch between newly analyzed atoms and persisted call relations.
- New runtime changes should continue preserving the atom's native ID format. No forced absolute-vs-relative conversion should be reintroduced in relation writers.
