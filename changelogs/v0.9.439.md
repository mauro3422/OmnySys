# v0.9.439 - Trust Batch Remediation and Issue Persistence Alignment

## Summary

This iteration adds a batch-based trust remediation layer to attack drifts in order, and aligns issue persistence across the integrity detector, the canonical snapshot, and the technical debt report.

## Key Changes

- Canonical batch plan: added a remediation batch report to compare snapshots before and after changes, with clear exit criteria for `issue_persistence`, `policy_drift`, `metadata_coverage`, `tool_latency`, and `folderization`.
- Unified issue persistence: the current metrics snapshot now exposes `watcherIssuePersistence`, and the technical debt report persists the same signal so reports no longer disagree when one surface says `orphans=0` and another says `orphanedIssues>0`.
- More faithful trust report: the trust report now separates watcher and pipeline orphans, uses both sources in the confidence score, and exposes remediation state for operational follow-up.
- CLI and MCP: the `trust` command prints remediation batches, and the new MCP trust report tool exposes the canonical state without relying on browser visualizations.
- Validation: updated tests and mocks for the technical debt report to reflect the real explainability flow and persisted snapshot data.
