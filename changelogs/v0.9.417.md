# v0.9.417

## Added
- Added a canonical system inventory layer so the compiler health stack can classify canonical surfaces, emergent systems, bridges, wrappers, and promotion candidates in one reusable report.
- Surfaced system inventory in the metrics snapshot, health dashboard, status payload, and status table so the runtime can show system centrality alongside tool inventory and cache policy.
- Added a dedicated `mcp_omnysystem_get_system_inventory_report` tool to expose the inventory without reconstructing it ad hoc from the status surfaces.

## Changed
- Reused the shared compiler snapshot context to propagate system inventory through metrics, health, and status consumers without rebuilding the same classification in each layer.
- Extended the compiler health dashboard and status table to show system inventory next to the existing tool inventory and cache policy views.
