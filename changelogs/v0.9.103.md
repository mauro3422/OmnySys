# Release v0.9.103 - Genetic Preservation Policy & System Stabilization

## Summary
This release establishes the **Genetic Preservation Policy**, migrating the entire pipeline from physical deletions to a canonical "Soft Delete" model. It also stabilizes the system after schema drift, achieving a **100/100 Health Score (Grade A)**.

## Key Achievements

1. **Genetic Preservation (Phase 2)**: 
   - Refactored `live-row-cleanup.js`, `orchestrator/issues.js`, and `watcher-issue-persistence.js` to use logical deletes (`is_removed = 1`).
   - Standardized historical record keeping for atoms, relations, and semantic issues.
2. **Schema Stabilization**:
   - Fixed `schema-registry.js` with robust `NOT NULL` migrations for SQLite.
   - Synchronized `updated_at` and `is_removed` columns across core tables.
3. **Watcher Contract Synchronization**:
   - Updated `watcher-issues.js` and `watcher-issue-lifecycle.js` to ensure the `lifecycle` metadata is persisted in the database.
   - Fixed `PipelineIntegrityDetector` failures related to missing issue metadata.
4. **Semantic API Hardening**:
   - Updated `SemanticQueryTool` and `semantic-queries.js` to filter out historical records by default, ensuring clean reporting while preserving the "Genetic Memory".
5. **Integrity Restoration**:
   - Reached **100/100 Health Score** after resolving orphaned data and incomplete metadata issues.
