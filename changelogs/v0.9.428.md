# v0.9.428

## Summary

This release surfaces folderized naming normalization across the compiler health stack so naming debt becomes visible in metrics, technical debt, and explainability views.

## Highlights

- Added a safe folderized-name normalizer that only executes through the existing rename flow when the plan is explicitly safe.
- Surfaced normalization summary data in folderization, compiler metrics, technical debt, and explainability snapshots.
- Added a technical debt priority action for folderized naming normalizer work so normalization is now an explicit maintenance item.
- Kept the normalizer non-intrusive: analysis and planning stay read-only, and execution only happens on approved folderized families.
