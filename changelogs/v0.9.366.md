# v0.9.366 - Sprint 16: Deferred Runtime Reload for Reindex-Worthy Changes

## Summary
- Changed the hot-reload coordinator so reindex-worthy runtime changes no longer trigger an immediate module reload.
- Reindex events now let the DB/indexing pass settle first and then queue a restart only when the runtime restart mode allows it.
- Preserved immediate reload behavior for refresh/reload-safe surfaces while keeping restart surfaces on the existing restart path.

## Validation
- Smoke import passed for the hot-reload manager and runtime change policy modules.
- The new policy path remains compatible with the existing restart coordinator and proxy-managed restart flow.
