# v0.9.146 - Sprint 15: Call Graph Preservation & Bootstrap Hardening

**Date:** 2026-03-19

## Summary

This release closes the loop on the bootstrap regression that was wiping canonical call relations during live-row reconciliation and crashing the MCP runtime through a stale barrel export.

## Key Achievements

1.  **Call Graph Preservation**: Disabled automatic live-row cleanup for `atom_relations` so canonical `calls` rows are no longer soft-deleted during startup reconciliation.
2.  **Bootstrap Crash Fix**: Removed the stale `getRecentCommits` barrel export from the file watcher handlers and moved the helper into a dedicated module.
3.  **Canonical Recovery**: Reactivated valid `calls` rows in SQLite so `call_graph` and `atom_relations` are aligned again after the recovery pass.
4.  **Health Validation**: Confirmed the runtime health summary returns `A+` with hydrated `call_graph` rows and no orphan call relations.
