# Changelog v0.9.394 - Incremental Update Surface & Repository Integrity

> **Fecha:** 2026-04-02
> **Tipo:** Hardening + observability

## Update pipeline

### Incremental mirror sync
- `saveFileResult()` now refreshes the file summary mirror and incremental system map in the same mutation path.
- The watcher analysis flow now persists the file result after parsing, so `files`, `system_files`, and `file_dependencies` stay aligned after each atom/function change.
- `used_by_json` is recomputed incrementally for touched surfaces so consumers do not depend on a global reindex to see the latest references.

### Status surface
- Added an explicit `Update` row to the control-plane terminal summary.
- The update summary now reports:
  - live files
  - file mirror rows
  - dependency totals
  - pending phase-2 files
  - watcher queue/failures
  - repository journal backlog
  - SQLite integrity state
- Mirrored the same update signal into the bootstrap dashboard so startup summaries expose the sync posture alongside phase-2 and health metrics.

## Repository integrity hardening

- Repository diagnostics now carry SQLite integrity probe results.
- `get_server_status()` can now degrade the database health surface if the repository integrity probe fails.
- Cache reuse now respects closed SQLite connections instead of assuming the singleton is still live.

## Validation

- Incremental system-map sync tests pass.
- File watcher persistence tests pass.
- Status table tests pass with the new `Update` surface.
