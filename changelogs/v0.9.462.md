# v0.9.462 - Compiler Observability Runtime Split and Live Status Freshening

Date: 2026-04-19

## Summary

This release moves the compiler observability contract onto a dedicated runtime entrypoint and reconnects the live status / health surfaces to the fresh path.

## What changed

- Added `src/shared/compiler/compiler-observability-contract-runtime.js` as the canonical runtime implementation for compiler observability.
- Redirected the shared compiler barrel to export the runtime entrypoint instead of relying on the older contract path.
- Updated compiler snapshot assembly and snapshot service flow to load the fresh runtime contract.
- Updated live `status` and `health` tool error surfaces to include the runtime stack when failures happen.
- Kept the old contract file aligned with the new state helper so stale live loads have fewer chances to fail on `normalizeState`.

## Result

- The live MCP surfaces now report through the fresh observability runtime path.
- The repo has a canonical runtime entrypoint for compiler observability, instead of a fragile split between old contract logic and live callers.
- The current worktree is ready to be published as a single coherent patch.
