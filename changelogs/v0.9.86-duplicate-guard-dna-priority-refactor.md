## Summary

Two targeted improvements to reduce false positives in the Duplicate Guard and eliminate
real code duplication in the priority calculation logic.

---

## What Changed

### 1) Duplicate Guard — DNA hash analysis + name whitelist

`src/core/file-watcher/handlers/file-handlers.js`

**Problem:** The guard previously reported any symbol with the same name in multiple files as a
duplicate, generating false positives for common JS function names like `processChange`,
`initialize`, `validate`, etc.

**Solution:** Two-level detection → **replaced** by unified DNA SQL query (same logic as `queryDuplicates`):

- **Single SQL query:** Gets the `dna_json` fingerprints of atoms in the changed file from SQLite,
  then queries for other atoms with identical DNA in other files (excluding test files).
- **Zero false positives:** If two functions have different DNA, they are never reported — regardless
  of their name. No whitelist needed.
- **Same engine as `aggregate_metrics(duplicates)`:** Both the watcher and the MCP tool now use
  identical detection logic — the only difference is scope (single file vs. full project).
- All findings are classified as `LOGIC_DUPLICATE` — name coincidences without matching DNA are
  silent by design.

Previous approach (name-first + `getAtomsByName` loop per symbol) was replaced entirely.

### 2) Priority Calculator — Unified to canonical module

**Problem:** Three independent implementations of change-priority logic existed:
1. `batch-processor/priority-calculator.js` → canonical, most complete
2. `orchestrator/helpers.js` → `_calculateChangePriority(change)` — simplified copy
3. `unified-server/initialization/batch-processor-init.js` → `calculateChangePriority(change)` — another copy

**Solution:** Migrated both consumers to import and use the canonical `calculatePriority`
from `batch-processor/priority-calculator.js`.

`src/core/orchestrator/lifecycle/watchers/file-watcher.js`
- Added import: `import { calculatePriority } from '../../../batch-processor/priority-calculator.js'`
- Replaced: `this._calculateChangePriority(change)` → `calculatePriority(change.filePath, change.changeType)`

`src/core/unified-server/initialization/batch-processor-init.js`
- Removed local `calculateChangePriority` function (~10 lines)
- Removed unused `import path from 'path'`
- Added import: `calculatePriority` from `../../batch-processor/index.js`
- Replaced: `calculateChangePriority(change)` → `calculatePriority(change.filePath, change.changeType)`

The canonical `calculatePriority` is richer: it also factors in `exportChanges`,
`dependentCount`, and `importChanges` — which the local copies ignored.
