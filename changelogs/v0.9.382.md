# v0.9.382 - Sprint 16: Worker-Aware Repository Mutations and Deferred Hash Flush

## Scope
- Make the write window for unified analysis more robust under SQLite contention.
- Prevent durable mutations from being enqueued into worker-local memory where they can be lost when the worker exits.

## Changes
- Deferred file-hash persistence in `src/layer-a-static/pipeline/unified-analysis.js` until all workers settle, so the parent process writes hashes in a single post-settlement window instead of racing the worker pool.
- Taught `src/layer-c-memory/storage/repository/repository-bridge.js` to treat durable mutations in worker threads as retryable inline work instead of queueing them into a worker-local journal.
- Kept worker-side bulk atom and file-summary writes visible by warning on explicit `success: false` results from the mutation bridge.

## Verification
- `node --check` passed for the touched repository bridge, unified analysis, and worker analysis modules.

- `vitest run tests/unit/layer-c-memory/mcp/core/mutation-settlement.test.js --config tests/config/vitest.unit.config.js` passed.
