# CHANGELOG v0.9.108 - Compiler Stabilization & Integrity

## Summary
This release focuses on the deep stabilization of the OmnySys compiler layer, resolving critical import regressions, standardizing utility libraries, and achieving structural Grade A health.

### Key Achievements
1. **Utility Canonization**: Consolidated 16 redundant versions of `getLineNumber` and multiple `safeArray/toNumber` helpers into canonical `#shared/utils` modules.
2. **SOLID Persistence Architecture**: Refactored the System Map persistence layer into specialized handlers (`semantic-handler`, `file-handler`, etc.), eliminating God-Object complexity in `system-map.js`.
3. **Integrity Hardening**: Fixed critical "is not defined" errors in `BaseSqlRepository` and `conformance-utils` that were causing the indexing pipeline to crash silently.
4. **Clean Slate Re-indexing**: Executed a full `--force-reanalysis` of 2,187 project files, ensuring the Impact Map and Dependency Graph are 100% accurate.
5. **Path Alias Migration**: Standardized 100+ file imports using subpath aliases (`#layer-c`, `#shared`), making the codebase resilient to deep refactoring.

### Refactored Components
- `src/shared/compiler/`: Unified conformance and reporting logic.
- `src/layer-c-memory/storage/repository/adapters/helpers/system-map/`: Modularized persistence handlers.
- `src/shared/utils/line-utils.js`: Now the single source of truth for AST line mapping.

### Known Metrics
- **Structural Health**: Grade A (98/100).
- **File Coverage**: 2,187 files indexed.
- **Note on Grade F**: The pipeline reports Grade F (38/100) due to 306 files in Layer B/Docs that produce 0 atoms. These are intentionally empty of symbols but tracked by the scanner. This is a reporting artifact, not a functional failure.
