# v0.9.164 - Sprint 16: Framework Lifecycle Hook Policy

**Date:** 2026-03-21

## Summary

This release teaches the duplicate-signal policy to recognize initialization rollback hooks and extraction-strategy confidence hooks as intentional framework contracts. Those methods are now treated like other lifecycle hooks instead of accidental conceptual duplicates, which lowers noise on the policy surface without weakening the signal on real business-logic duplication.

## Key Changes

1. **Initialization Lifecycle Policy**: `rollback` hooks under the initialization pipeline are now treated as intentional framework lifecycle methods.
2. **Strategy Hook Policy**: `calculateConfidence` hooks in the type-contract strategy hierarchy are now treated as intentional framework contract methods.
3. **Policy Refactor**: the conceptual duplicate decision path was compacted into smaller helpers so the policy stays under the complexity threshold.
4. **Regression Coverage**: the policy test now covers framework and non-framework cases for `performAction`, `rollback`, and `calculateConfidence`.

## Validation

- `node --check` passes for the duplicate-signal policy detector.
- The policy regression test passes under Vitest.
- The runtime watcher now only shows the policy module as stale until the next reload; no new active alerts were introduced.
