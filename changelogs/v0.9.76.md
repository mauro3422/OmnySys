# v0.9.76 — Remove broken ESM hot-reload from FileWatcher

**Release date:** 2026-03-03

## Overview

Removed the broken ESM module-reloading behavior from the hot-reload system.
Node.js ESM caches all modules permanently within a process — there is no way
to unload a module without restarting the process. The previous implementation
attempted workarounds (`import(url + '?bust=timestamp')`, `refreshToolRegistry`)
that silently failed due to parent modules holding stale function references.

**FileWatcher is preserved** — it continues to detect file changes and notify
the orchestrator to re-index atoms in SQLite. That functionality is correct and
necessary.

To apply code changes to MCP tools: restart the VS Code task "OmnySys MCP Daemon" (~8 seconds).

## Changes

### `strategies/tool-strategy.js`
- Removed `import(url + ?bust=...)` call
- Removed `refreshToolRegistry()` call
- Now only logs: `"Tool changed — restart task to apply (8s)"`
- Added clear docstring explaining WHY module reload is not done

### `strategies/pipeline-strategy.js`
- Removed `import.meta.resolve()` + `import(uniqueImport)` attempt
- Removed `server.emit('hot-reload:pipeline-updated')` (no longer invalidates cache)
- Now only logs: `"Pipeline module changed — restart task to apply (8s)"`

### `hot-reload-manager/index.js`
- Updated module description: renamed from "Hot Reload Manager" to "File Change Watcher Manager"
- Added `⚠️ IMPORTANT` section documenting what the system does NOT do
- Version bumped to 2.1.0

## Preserved

- `restart_server` tool still calls `refreshToolRegistry()` — this is a manual,
  explicit call initiated by the user. It's not the automatic watcher.
- `FileWatcher` + `ModuleClassifier` unchanged — atom re-indexing is unaffected.
- All 4 stub strategies (`ExtractorStrategy`, `HandlerStrategy`, `QueryStrategy`,
  `LifecycleStrategy`) unchanged — they were already no-ops.

## Future

A plugin system for user-defined tools (files in a `plugins/` folder outside the
main module graph) could support true hot-loading via isolated dynamic imports,
without touching the server's own ESM module cache.
