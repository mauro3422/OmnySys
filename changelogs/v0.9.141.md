# v0.9.141 - Sprint 14: Action API Standardization & Telemetry Unification

## Overview
This release focuses on architectural stabilization, unifying how the system executes actions and reports telemetry. It also implements critical noise reduction for test-related findings.

### Key Achievements

1.  **Action API Standardization**: Created `ActionEngine.js` as a Single Source of Truth (SSOT) for executing system mutations, refactorings, and test generation. All MCP action tools now delegate to this engine.
2.  **Telemetry Unification**: Migrated multiple core modules (`ValidationEngine`, `RuleRegistry`, `GuardRegistry`, `Composer`) to the central `StatsPool`. Ad-hoc `getStats` methods have been removed in favor of a provider-based registration pattern.
3.  **Noise Reduction**: Implemented a global filter in `core-policy.js` to ignore findings from generated tests (fingerprint: `test_framework:generated`), resulting in a cleaner and more actionable health score.
4.  **Recommendations Engine Consolidation**: Refactored multiple utility modules to use the consolidated `RecommendationEngine.js` for architectural suggestions.
5.  **System Integrity**: Achieved and verified a **100/100 Health Score** (Grade A+) with all 8 pipeline integrity checks passing.
