# v0.9.75 — MCP Tools Bugfixes & HTTP Proxy

**Release date:** 2026-03-03

## Bug Fixes

### `suggest_refactoring` — Returned 0 suggestions (critical)
- **Root cause 1:** `getAllAtoms` filter used `a.is_dead_code` (snake_case) but `rowToAtom` maps the field as `a.isDeadCode` (camelCase). Dead code check was silently skipped, but all atoms still passed... yet downstream `extract-analyzer` had broken logic.
- **Root cause 2:** `identifyLogicalBlocks` used `t.operation` (field doesn't exist in actual `transformations` shape). Actual shape is `{type, kind, line, to, from}`. Field corrected to `t.type`.
- **Root cause 3:** `identifyLogicalBlocks` never initialized `currentBlock.endLine`, making the gap condition `t.line - undefined > 5` always `NaN` → always false → 1 block → not enough to suggest extraction.
- **Fix `atom.js`:** Changed filter to `!a.isDeadCode` (camelCase).
- **Fix `extract-analyzer.js`:** Corrected `identifyLogicalBlocks` field names and initialized `endLine`. Added LOC+complexity-based analysis (LOC > 80 && complexity > 5, or complexity > 15 && LOC ≤ 80) as primary signal — 305 functions qualify.

### `get_schema` — Showed `totalAtoms: 0`
- Same `isDeadCode` camelCase fix in `getAllAtoms` resolves inventory showing all zeros.

### Hot-reload — Storage layer changes not triggering `refreshToolRegistry`
- **`module-classifier.js`:** Added `storage/atoms/` and `storage/repository/` to hot-reload reloadable patterns. Changes to `atom.js`, `converters.js`, etc. now trigger `refreshToolRegistry` automatically.

### `restart_server` — `refreshToolRegistry` silently failed
- **`restart-server.js`:** Changed static `import('../mcp-http-server.js')` to a cache-busted `?bust=timestamp` URL. Errors are now logged at `warn` level instead of being swallowed silently.

## New Features

### HTTP Proxy (`mcp-http-proxy.js`) — experimental
- New `src/layer-c-memory/mcp-http-proxy.js` wraps `mcp-http-server.js` as an IPC child process.
- When `restart_server` is called under the proxy, the worker is killed and respawned with a fresh ESM cache — **all code changes reload without IDE restart**.
- `npm run mcp:http:proxy` script added.
- **Note:** Currently marked experimental. HTTP sessions drop on worker restart (Antigravity reconnection requires IDE restart). Not used as default task — proxy task available in `.vscode/tasks.json` for future use once session continuity is solved.

### On-demand Phase 2 prioritization (`static-insights.js`)
- `_prioritizeFileForPhase2()` triggers deep metadata analysis for a specific file on-demand when queried by MCP tools, ensuring up-to-date `dataFlow`, `errorFlow`, and semantic data.

### Phase 2 verbosity control (`unified-analysis.js`)
- Background Phase 2 indexer now passes `verbose=false` to `BatchTimer`, silencing nested progress bars in the terminal.
