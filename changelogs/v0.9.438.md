# v0.9.438 — MCP Bridge Hardening & Summary Coherence Repair

## Resumen

Esta iteracion refuerza la integracion MCP entre Windows, WSL y Codex, y vuelve canonica la generacion de summaries para que lo que se persiste, observa y reporta salga del mismo origen.

## Cambios Clave

- **Bridge MCP mas robusto**: el daemon HTTP vuelve a escuchar de forma mas flexible para clientes locales y el stdio bridge deriva sus URLs desde helpers compartidos.
- **Config WSL mas estable**: la sincronizacion de Codex/WSL deja de reescribir URLs de forma ambigua y conserva un bridge consistente con route base explicita.
- **Summary coherence**: se agrego `buildCompilerMetricsSnapshotSummary()` como fuente canonica y `validateSnapshotSummaryCoherence()` para detectar drift entre summary persistido y snapshot derivado.
- **Observability y persistencia alineadas**: los snapshots de compilador ahora persisten el summary canonico y exponen el estado de coherencia en los contratos de observabilidad.
