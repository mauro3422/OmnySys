# Changelog v0.9.404 - Tool Cache Policy and Helper De-duplication

> **Fecha:** 2026-04-03
> **Tipo:** Observability cleanup + helper consolidation

## Tool cache policy

- Classified the heavy MCP tools into explicit cache tiers: `live`, `ttl-cache`, `snapshot-cache`, and `fingerprint-cache`.
- Marked `get_metrics_snapshot`, `query_graph`, `traverse_graph`, `get_technical_debt_report`, `check_pipeline_integrity`, and `diagnose_tool_health` as fingerprint-derived surfaces.
- Kept `get_server_status`, `get_recent_errors`, and `execute_sql` as live gates because they are freshness-sensitive.
- Surfaced the cache tier summary in the status panel so the terminal shows the current policy directly.

## Helper de-duplication

- Moved shared `takeSample` logic into a single canonical helper and reused it from the metrics, dashboard, cache policy, and explainability summary layers.
- Renamed the explainability watcher compact helper so it no longer collides with the status-plane watcher compactor.
- Re-exported the compact snapshot helpers through `summary.js` as a thin barrel.

## Validation

- `get_technical_debt_report()` now reports `0` structural debt and `0` conceptual debt after the helper consolidation.
- `get_recent_errors()` remains clean after the refactor.
- The core compiler dashboard and telemetry tests still pass after the cache-tiering changes.
