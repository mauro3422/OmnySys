# v0.9.148 - Sprint 15: Relation Cascade Fix for File Deletions

**Date:** 2026-03-19

## Summary

This hotfix closes the remaining source of orphan call relations: deleting a file or atom now soft-deletes the call relations that point to or originate from the removed atoms.

## Key Changes

1.  **Cascade on Atom Deletion**: `deleteAtomRecord()` now also soft-deletes `calls` relations for the removed atom.
2.  **Cascade on File Deletion**: `deleteByFile()` now gathers all atom ids for the file and removes any `calls` relations attached to them before the file is considered gone.
3.  **Drift Prevention**: File-watcher and cache cleanup paths now funnel through the same relation-cascade behavior, so deleted atoms no longer leave dangling call edges behind.

## Notes

- This fix addresses the recurring orphan-call drift without relying on manual DB cleanup.
- Existing orphaned rows from prior runs still need one reconciliation pass or a fresh reindex to clear the old state from SQLite.
