# v0.9.78 — SQL Atom Enrichment & OmnySys Self-Optimization

**Release date:** 2026-03-03

## Overview

Added deep SQL parsing and enrichment to OmnySys. The system now fully understands its own DB accesses, detecting SQL injection risks, extracting schema drift vulnerabilities via AST, and tracking SQL-to-JS relationships. Built an OmnySys-specific health detector that catches repository bypasses and query inefficiencies.

## Key Features

### 1. Advanced SQL Atom Extraction
- **Tree-sitter AST Walk:** `sql.js` extractor now precisely walks the `@derekstride/tree-sitter-sql` AST to extract `tables_referenced` and `columns_referenced`.
- **Regex Fallback:** Added `detectOperationFallback` for complex queries (`PRAGMA`, `WITH`, `DDL`) that tree-sitter misses.
- **SQL Variable Resolver:** Template literal variables (`${var}`) are now analyzed to detect `sql_injection_risk` based on user-input heuristics (req, res, ctx) and resolved against local file constants.

### 2. Semantic SQL Fields
Every SQL atom now emits rich metadata:
- `sql_purpose`: 16 new semantic categories (`AGGREGATION`, `CACHE_READ`, `BULK_MUTATION`, `SQL_INJECTION_RISK`, `DYNAMIC_QUERY`, etc.).
- `parent_atom_id` / `parent_atom_name`: SQL atoms accurately link to their parent JS function using line-range bounding.

### 3. Graph Integration
- `build-links.js` now persists an `executes_sql` relation natively in the `atom_relations` table connecting the parent JS atom to the SQL atom.
- The `query_graph` MCP tool can now traverse from a JS function directly to the DB queries it executes.

### 4. Pattern Detection (Auto-Optimization)
- **`sql-patterns-detector`:** Flags high query density functions, N+1 risks (SQL inside complex functions), and dead SQL.
- **`omnysys-health-detector`:** A new bespoke detector analyzing OmnySys's own SQLite usage:
  - **`sql-repo-bypass`:** Flags direct DB access (`db.prepare()`) outside the `storage/` layer (respecting legitimate bypasses like `cache/` and `file-watcher/`).
  - **`sql-join-candidate`:** Detects JS functions that execute multiple sequential `SELECT`s on different tables that could be unified.
  - **`sql-schema-drift`:** Accurately compares AST-extracted query columns against the canonical `schema-registry.js`, catching runtime bugs where queries reference non-existent columns.
  - **`sql-dynamic-in-storage`:** Flags template-literal based queries in the core DB layer that should be static.

## Chore & Fixes
- **`project-query.js`:** Optimized `getProjectMetadata()` by combining 4 separate `atoms` table queries into 2 via derived JS counts, resolving a `sql-join-candidate` finding.
- **`semantic-handler.js`:** Fixed a bug where `connection_key` and `weight` columns were not being persisted on insert despite existing in the schema.
