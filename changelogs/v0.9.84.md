# v0.9.84 - Impact Atomic Simulator & Schema Discovery

## Visión General
Esta versión introduce herramientas predictivas de alto impacto para la refactorización arquitectónica. Se enfoca en resolver las limitaciones del análisis estático superficial, permitiendo al sistema y al Agente responder con precisión quirúrgica a las consecuencias de mutaciones en el código central.

## ✨ Nuevas Capacidades (Layer C - MCP)
- **Simulador de Impacto Atómico (`mcp_omnysystem_impact_atomic`):**
  - Desarrollamos un simulador predictivo que mapea el "radio de explosión" exacto de alterar una función o clase (átomo).
  - En lugar de rastrear dependencias a nivel de archivo (como el antiguo *Impact Map*), utiliza una **Common Table Expression (CTE) Recursiva en SQLite** sobre la tabla `atom_relations` para iterar infinitamente hacia arriba el árbol de llamadas.
  - Soportando `intents` como `signature_change` y `deletion`, la herramienta realiza un `LEFT JOIN` a altísima velocidad evitando el cuello de botella tradicional de hidratación en memoria (Node.js/V8).
  
- **Auto-descubrimiento Activo de Esquemas SQL:**
  - Se potenció `execute_sql` para inyectar automáticamente `schemaHints` (mediante `PRAGMA table_info`) como mecanismo de auto-corrección instantánea cuando el Agente intenta consultar columnas inexistentes.
  - Consolidamos la postura táctica de que el acceso a RAW SQL es un superpoder suficiente para explorar esquemas orgánicamente bajo demanda, evitando saturar la ventana de contexto.

## 🛠 Refinamientos Arquitectónicos
- Se actualizó el *File Watcher* (`impact-wave.js`) manteniendo su análisis superficial ágil (profundidad 1) para cuidar el rendimiento en tiempo real (save/edit), promoviendo que el CTE Recursivo profundo quede reservado exclusivamente para la herramienta manual de refactorización pesada.
- Refactorización fina en el ruteo de `index.js` para registrar las nuevas Action Tools correctamente dentro del bus MCP.

## 📊 Impacto de Productividad
- A partir de esta versión, el sistema puede predecir exactamente cuántos átomos se romperán por un cambio de firma antes de escribir la primera línea de código y compilar.
