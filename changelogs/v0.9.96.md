# v0.9.96 - Canonical Persistence Sync & Runtime Drift Reduction

**Fecha:** 2026-03-06  
**Tipo:** Architecture + Compiler Standardization  
**Impacto:** Alto

---

## Resumen

Esta versión cierra una tanda de deuda técnica enfocada en unificar entry points canónicos, separar runtime restart de sync de datos, y bajar deriva entre SQLite, watcher, MCP y políticas del compilador.

El resultado fue un sistema más coherente para agentes: menos lógica duplicada en tools/runtime, menos falsos positivos en detectores de adopción y una superficie canónica más clara para sincronización, persistencia y evaluación.

---

## Cambios Principales

### 1. Sync Canónico de Filas Live/Stale

- Se agregó `ensureLiveRowSync()` como entry point de reconciliación en `shared/compiler`.
- `get_schema(database)`, `risk-query` y `pipeline-health` ahora consumen la sync canónica antes de reportar drift.
- El cleanup de support tables pasó de manual/diagnóstico a auto-remediación segura con degradación controlada bajo locks SQLite.

### 2. API Canónica de Evaluación

- Se consolidó `atom-evaluation.js` con evaluación compartida de:
  - `testability`
  - `semantic_purity`
  - señales de refactorización
- `suggest_refactoring` y sus analizadores principales dejaron de reconstruir heurísticas locales y ahora leen `compilerEvaluation`.
- Las reglas de conformance reconocen esta adopción como válida.

### 3. Runtime Restart Manual y Freshness Real

- El hot reload del daemon ahora separa:
  - actualización de DB/cache/análisis
  - restart real del runtime
- Se agregó modo manual de restart runtime para evitar reinicios automáticos excesivos del worker.
- `status` y `restart_server` exponen mejor el estado de restart pendiente.

### 4. Bridge Canónico de Persistencia del Compilador

- Se agregó `compiler-persistence.js` con helpers compartidos para:
  - `hasPersistedCompilerAnalysis`
  - `getPersistedIndexedFilePaths`
  - `findIndexedFileCandidate`
  - `cleanupOrphanedCompilerArtifacts`
  - `removePersistedFileMetadata`
  - `removePersistedAtomMetadata`
  - `emitOrphanedImportsFromPersistedMetadata`
- `cleanup`, `metadata-cleanup`, `file-scanner` y `fix-imports` dejaron de abrir fs + repo logic inline.

### 5. Detectores Más Honestos

- `service-boundary-conformance` ahora ignora strings/comentarios y detecta llamadas reales.
- `canonical-extension-conformance` dejó de marcar cualquier `classify*` local como superficie canónica paralela.
- `async-error-conformance` reconoce mejor contratos de recuperación delegada y dejó de disparar ruido por comentarios/strings.

### 6. Limpieza de Runtime y Watcher

- Se endureció `shutdown.js` con un boundary explícito de lifecycle.
- Se limpió reconciliación de watcher alerts stale/orphaned en almacenamiento y persistencia.
- `get_schema` dejó de usar una taxonomía vieja de átomos y ahora refleja `atom_type` real de SQLite.
