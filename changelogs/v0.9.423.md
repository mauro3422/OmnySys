# v0.9.423

## Bridge Runtime Telemetry

- Added canonical bridge runtime telemetry so the MCP stdio bridge can surface client disconnects, transport closes, session expiry, and reconnect loops in status and health.
- Preserved proxy and bridge runtime telemetry across restarts so the runtime no longer loses the history needed to tell fresh boot noise from a real thrashing pattern.
- Surfaced bridge runtime telemetry in the system table, structured status payload, and compiler metrics snapshot so the client transport state is visible alongside proxy health and startup telemetry.
