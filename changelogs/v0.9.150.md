# v0.9.150 - Sprint 15: Canonical ID Helper Centralization

**Date:** 2026-03-19

## Summary

This refactor centralizes canonical atom ID normalization and variant generation into a shared helper and adds explicit async error boundaries around shared-state linkage paths.

## Key Changes

1.  **Shared Canonical IDs**: `normalizeCanonicalAtomId` and `buildCanonicalAtomIdVariants` now live in a single helper module reused by the relation writers and pipeline linker.
2.  **Async Boundaries**: Shared-state linkage paths now catch and log failures instead of surfacing uncaught async errors.
3.  **Deduplication Cleanup**: The canonical ID logic is no longer duplicated across the relation adapters, reducing conceptual noise in the watcher.
4.  **Release Tracking**: Bumped the release index to reflect the current canonical ID and runtime safety work.

## Notes

- The live database remains aligned; this release is about keeping the codepath aligned with it.
- A runtime restart is still required after this refactor to load the updated linker module.
