# v0.9.185 - Sprint 16: Canonical Smart Batch Processor Factory

## What changed
- Added a canonical `createSmartBatchProcessor` factory so the file-watcher initializer no longer instantiates the processor class directly.
- Re-exported the factory through the batch-processor barrels for compatibility.
- Kept the existing `SmartBatchProcessor` class intact while routing the canonical creation path through the factory.

## Why
- `SmartBatchProcessor` was being flagged as a pipeline orphan even though it was still part of the file-watcher boot path.
- Giving the runtime a named creation helper makes the constructor path more explicit and easier for the graph to classify.
- This preserves the current behavior while reducing semantic ambiguity in the watcher pipeline.

## Validation
- `node --check` passed for the batch processor module, the barrels, and the initializer.
- `get_recent_errors()` only reported the expected restart requirement for `src/core/file-watcher/lifecycle/initialization.js`.
