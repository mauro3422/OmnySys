# v0.5.2 - Folderization Pipeline & Split-Large-File Integration

## 🚀 Features

### split_large_file: Folderización Automática
- **Nueva funcionalidad**: `split_large_file` ahora crea una **estructura folderizada** en lugar de archivos con sufijos
  - Antes: `file-public.js`, `file-helpers.js`
  - Ahora: `file/` con `index.js`, `public.js`, `helpers.js`
- **Extracción de código real**: Los archivos generados contienen el código original extraído por metadatos de átomos
- **Análisis de imports**: Detecta qué imports realmente usa cada grupo y los incluye automáticamente
- **Ajuste de paths**: Los imports relativos se ajustan para la nueva ubicación en subcarpeta
- **Eliminación de imports circulares**: Detecta y elimina imports que apuntan al propio archivo

### detect_folderization_opportunities: Fix SQLite
- **Fix REVERSE()**: Reemplazado uso de `REVERSE()` (no soportada en SQLite) con helpers JavaScript
- **Extracción por JavaScript**: Las queries de folderización y naming debt ahora usan extracción por JavaScript en lugar de SQL complejo

## 🐛 Bug Fixes

### syntax-validator.js
- **Fix require en ES module**: El syntax validator ahora pasa `spawnSync` como parámetro en lugar de usar `require()` que no está disponible en contexto ES module

### directory-structure-folderization-analysis-helpers.js
- **Recreación del archivo**: El archivo fue recreado con implementaciones reales de `buildFamilyStateReport`, `buildFolderizationCandidateReport` y `summarizeFamilyEvolution`

## 🔧 Technical Improvements

### split-large-file-helpers.js
- **Nuevo `buildFileContent`**: Extrae código real del archivo original usando metadatos de átomos
- **Nuevo `findImportsForGroup`**: Analiza qué imports realmente usa cada grupo basándose en símbolos
- **Nuevo `extractRealAtomCode`**: Extrae líneas reales usando `line_start/line_end`
- **Nuevo `adjustImportPath`**: Ajusta imports relativos para subcarpetas y elimina circulares
- **Nuevo `buildSplitPlan`**: Genera paths folderizados en lugar de sufijos
- **Nuevo `executeSplitPlan`**: Crea archivos split primero, barrel después (orden correcto)

### split-large-file.js
- **Actualización de `executeSplitPlan`**: Crea estructura folderizada completa (barrel index.js + archivos split)
- **Eliminación de archivo original**: El archivo original se elimina después de la folderización

### Limitación conocida
El split_large_file genera archivos con **dependencias entre grupos**. Las funciones exportadas dependen de funciones internas que están en el otro archivo. Para una división completa se necesita incluir funciones dependientes o usar `traverse_graph` para detectar dependencias cruzadas.
