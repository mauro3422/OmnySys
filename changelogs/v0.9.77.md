# v0.9.77 — Phase 2 On-Demand Lazy Indexing (query_graph details)

**Release date:** 2026-03-03

## Overview

Fixed the Phase 2 on-demand trigger in `query_graph details`. When a symbol
is queried via `query_graph { queryType: 'details' }` and its file has not yet
been deeply analyzed (`isPhase2Complete = false`), the system now correctly
triggers `analyzeSingleFile` for that file before returning the response.

This enables a **lazy indexing pattern**: Phase 1 (static extraction) runs on
all files at startup, and Phase 2 (deep semantics) runs on-demand when a
specific file is actually queried.

## Changes

### `graph-query-tool.js`
- `execute()` now stores `this.projectPath = context.projectPath`
- This makes `projectPath` accessible throughout all subclass tool methods

### `query-graph.js` (details query)
- Fixed: `this.context?.projectPath` → `this.projectPath` (was always undefined)
- Added `Promise.race()` with 5s timeout so on-demand Phase 2 never blocks
  the tool response indefinitely
- If Phase 2 times out or fails, returns Phase 1 data with a `warn` log
- Improved log messages: `[Phase2:OnDemand]` prefix for easier debugging

## Flow

```
query_graph { queryType: 'details', filePath: 'x.js', symbolName: 'foo' }
   └─ getExactAtom('foo', 'x.js')
       ├─ atom.isPhase2Complete === true  → return immediately
       └─ atom.isPhase2Complete === false
           └─ analyzeSingleFile(projectPath, 'x.js', { verbose: false })
               └─ [5s timeout guard]
               └─ getExactAtom('foo', 'x.js')  ← enriched atom
               └─ return details
```
