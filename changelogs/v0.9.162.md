# v0.9.162 - Sprint 16: Barrel Surface Governance

**Date:** 2026-03-21

## Summary

This release teaches the compiler governance layer to distinguish pure barrels from mixed barrels that also carry local logic. The new policy keeps re-export entrypoints lean, pushes implementation behavior into dedicated modules, and gives the contract layer a dedicated metric for mixed barrel drift.

## Key Changes

1. **Mixed Barrel Policy**: `canonical-extension-conformance` now flags re-export modules that also define local logic as `local_barrel_with_logic`.
2. **Canonical Guidance**: the reuse guidance and architectural recommendation layers now map mixed barrels to a pure-barrel split action.
3. **Contract Layer Split**: the compiler contract layer moved surface/invariant builders into a dedicated helper module so the orchestrator stays thin and extensible.
4. **Regression Coverage**: a new unit test validates the mixed-barrel rule, the pure-barrel case, and the governance metrics count.

## Validation

- The updated compiler policy passes `node --check`.
- The new regression test passes under Vitest.
- The runtime is fresh after cache reload; the remaining watcher signal is only a low-impact structural alert on `compiler-contract-layer.js`.
