# v0.9.145 - Sprint 15: Canonical DB Enforcement

**Date:** 2026-03-18

## Summary

This release removes runtime fallback reads from persisted JSON relation fields and forces canonical reads through DB relations and APIs.

## Key Changes

1. Removed fallback usage of `calls_json` and `called_by_json` from runtime readers that drive guards, health checks, and semantic analysis.
2. Rewired pipeline integrity checks to validate `atom_relations` and active atoms instead of parsing stored JSON snapshots.
3. Kept JSON columns as legacy compatibility surfaces only, not as canonical sources of truth.
4. Fixed a relation-sample integrity query that was referencing a non-existent `atom_relations.file_path` column.

## Notes

- `calls_json` / `called_by_json` remain in schema for compatibility and serialization, but the system should no longer depend on them for canonical decisions.
- Restart the MCP runtime after this release so the updated modules are loaded.
