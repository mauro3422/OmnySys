# v0.9.94 - File Watcher Guard System v2.0 & Error Handling Detection Fix

**Fecha:** 2026-03-05  
**Tipo:** Feature + Bug Fix  
**Impacto:** Alto - Sistema de guards estandarizado y detector de errores mejorado

---

## 🎯 Resumen

Esta versión introduce el **File Watcher Guard System v2.0** completamente estandarizado, con 11 guards activos, sistema de metadatas unificado, y un **fix crítico** en el detector de error handling que ahora detecta correctamente try/catch en el código.

---

## ✨ Nuevas Características

### 1. Guard Standards System (`src/core/file-watcher/guards/guard-standards.js`)
- Sistema unificado de estándares para todos los guards
- `IssueDomains`: code, arch, sem, runtime, perf
- `StandardThresholds`: Complejidad, impacto, shared state
- `StandardSuggestions`: Mensajes de acción predefinidos
- `createIssueType()`: Nomenclatura estandarizada `{domain}_{subdomain}_{severity}`
- `createStandardContext()`: Contexto consistente con `suggestedAction`

### 2. Nuevos Guards (6 añadidos)

#### Semantic Guards
- **`async-safety-guard`**: Detecta funciones async sin error handling
- **`complexity-guard`**: Monitorea complejidad ciclomática y tamaño
- **`event-leak-guard`**: Detecta memory leaks por event listeners
- **`dead-code-guard`**: Alerta de código muerto (con filtros anti-falsos-positivos)

#### Impact Guards  
- **`hotspot-guard`**: Detecta funciones con cambio frecuente
- **`pipeline-health-guard`**: Monitorea shadow volume y zero atoms

### 3. Registry Mejorado (`src/core/file-watcher/guards/registry.js`)
- Registro con metadata (domain, version, description)
- Métricas de performance por guard
- Validación de guards al registrar

## 🔧 Mejoras

### Guards Estandarizados (5 actualizados)
- **`shared-state-guard`**: Ahora usa `createStandardContext()` y `createIssueType()`
- **`integrity-guard`**: Estandarizado con severidad calculada
- **`duplicate-risk-guard`**: Issue types estandarizados `code_duplicate_{severity}`
- **`impact-wave-guard`**: Estandarizado con contexto completo
- **`circular-guard`**: Issue types estandarizados `arch_circular_{severity}`

### Detector de Error Handling Fix (`src/layer-a-static/pipeline/phases/atom-extraction/builders/metadata-builder.js`)
**Problema:** El detector anterior solo buscaba strings `'try'` o `'catch'`, fallando en muchos casos.

**Solución:** Nueva función `detectErrorHandling()` con 4 patrones:
1. Bloque try/catch: `try { ... } catch ... {`
2. Promise `.catch()`: `.catch(`
3. Inline try/catch
4. Async/await con try

**Resultado:** 
- Antes: ~100 funciones detectadas con error handling
- Después: **471 funciones** correctamente detectadas

### Purpose Detection Mejorado (`src/layer-a-static/pipeline/phases/atom-extraction/metadata/purpose.js`)
- Nuevos propósitos: `ENTRY_POINT`, `WORKER_ENTRY`, `SERVER_HANDLER`, `POTENTIALLY_UNUSED`
- `checkEntryPoint()`: Detecta main/start/run/init
- `checkServerHandler()`: Detecta handlers en archivos server/proxy
- Cambio: `DEAD_CODE` → `POTENTIALLY_UNUSED` para reducir falsos positivos

### Bug Fix Crítico (`src/core/file-watcher/index.js`)
**Problema:** `this.pendingChanges` no inicializado causaba `Cannot read properties of undefined (reading 'set')`

**Fix:** Agregado en constructor:
```javascript
this.pendingChanges = new Map();
this.fsWatcher = null;
this.batchProcessor = null;
```
