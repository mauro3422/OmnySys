# Changelog v0.9.106 - SQL Handler Refactoring & Health Recovery

## Goal
Resolve critical code duplication in persistence handlers and restore system health metrics (Grade D/C recovery). Centralize SQL logic into generic repository methods to eliminate redundant boilerplate.

## Key Changes

### Core Persistence
- **`BaseSqlRepository`**: 
  - Implemented `loadTableRows` genéric loading utility with optional mapping.
  - Implemented `saveTableRows` batch UPSERT utility with transaction support.
  - Standardized `clearTable` and basic CRUD operations.
  - This centralization eliminated ~350 lines of redundant SQL boilerplate across handlers.

### System Map Handlers
- **`semantic-handler.js`**: Refactored `saveSemanticData`, `loadSemanticConnections`, and `loadSemanticIssues` to use generic repository methods.
- **`risk-handler.js`**: Replaced manual SQL preparation and loops with `saveTableRows` and `loadTableRows`. Integrated `BaseSqlRepository` for better structure.
- **`system-map.js`**:
  - Refactored `updateSystemMetadata` and `loadSystemMetadata` to utilize the new repository patterns.
  - Reduced `persistSystemMapToDb` cyclomatic complexity by extracting `normalizeConnections` and `normalizeIssues` helpers.
