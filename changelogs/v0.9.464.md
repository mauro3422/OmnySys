# v0.9.464 - Repository Bridge Canonicalization and Factory Path Normalization

Date: 2026-04-20

## Summary

This release collapses the repository bridge back into a canonical barrel, keeps the bridge state in the folderized source of truth, and normalizes repository factory project paths through the shared utility.

## What changed

- Replaced the root `repository-bridge.js` implementation with a canonical re-export of the folderized bridge barrel.
- Re-exported bridge state from `repository-bridge-state.js` so the public contract stays stable while the implementation lives under `bridge/`.
- Kept repository path normalization centralized in `repository-path-utils.js` for consistent factory lookups.
- Removed the last brittle default-export coupling from the repository bridge family.

## Result

- The repository bridge loads cleanly after a manual restart.
- `status`, `debt`, and recent error telemetry are clean again.
- The bridge family is now easier to evolve without reintroducing import drift.

## Notes

- This is a low-risk patch release focused on bridge hygiene and contract stability.
- The broader folderization/naming debt is still present in the project report, but no active bridge alerts remain on this path.
