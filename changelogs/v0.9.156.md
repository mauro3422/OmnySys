# v0.9.156 - Sprint 15: DB-Only Import and Export Validation

**Date:** 2026-03-20

## Summary

This release removes the remaining filesystem/runtime fallback paths from import and export validation. The watcher and validation tools now resolve against the canonical compiler DB only and fail explicitly with `DB_MISSING` when a file is not indexed.

## Key Changes

1.  **DB-Only Import Validation**: `validate-imports` now resolves broken imports and circular dependencies through canonical DB APIs only.
2.  **DB-Only Export Chain Validation**: `validate-exports-chain` now traces export and re-export chains from canonical file analysis and export metadata.
3.  **Explicit Missing-DB Failures**: files not present in the compiler index now return `DB_MISSING` instead of falling back to filesystem/runtime inspection.
4.  **Watcher Signal Cleanup**: the file watcher now reports export-chain validation from the canonical tool path, so stale runtime alerts are easier to reconcile after reload.

## Validation

- Syntax checks passed for `validate-imports.js`, `validate-imports/filesystem-validation.js`, and `validate-exports-chain.js`.
- Circular dependency resolution now walks `getFileDependencies(...)` recursively instead of using direct SQL.
- The remaining runtime watcher alerts are structural/stale until the MCP reloads the updated modules.
