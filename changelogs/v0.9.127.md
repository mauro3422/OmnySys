# Release v0.9.127 - Semantic Extractor Precision Fix

**Date:** 2026-03-17
**Scope:** Fix semantic analyzer generic unknown entities classification.

## Executive Summary

This patch release fixes a major logic flaw in the Semantic Analyzer component. Pure verbs (`build`, `create`, `execute`, etc.) were indiscriminately categorizing thousands of implementations as `unknown` unless their class or filename specifically ended in `Builder`. This caused significant noise in the conceptual duplicate reports, inflating the technical debt backlog with false positives.

## Major Changes

- **[FIX]** Removed strict `Builder` suffix requirement from `deriveEntity` strategy 2 and 3 in `semantic-analyzer.js`.
- **[ENHANCEMENT]** Expanded the list of cleanly stripped suffixes (`Manager`, `Service`, `Controller`, `Handler`, etc.) enabling the extractor to correctly identify core entities like `user` from contexts like `UserService.create` or `user-service.js`.

## Metrics & Impact (Post-Reindex)

- **Total Implementations Actionable:** Dropped from **999 to 694**.
- **Phantom Noise Eliminated:** The false generic groups (`build:logic:core:unknown`, `create:logic:core:unknown`, `execute:orchestration:core:unknown`), which accounted for roughly 232 implementations, have been resolved and correctly classified under their actual entities.
- **Improved Focus:** The True High-Priority Actionable groups in the pipeline (such as `process:orchestration:core:main` and `process:logic:core:with_file`) are now clearly visible at the top of the Technical Debt Report without being overshadowed by extraction artifacts.
