# Changelog v0.9.396 - Canonical Tool Inventory Subgroups & Ownership Cleanup

> **Fecha:** 2026-04-03
> **Tipo:** Hardening + observability

## Tool inventory refinement

### Canonical inventory policy
- Moved the tool inventory snapshot/report builder into `shared/compiler` so the MCP layer no longer owns the inventory policy locally.
- Introduced canonical subgroup classification for query, action, and admin tool families.
- Preserved the existing category buckets while adding a more useful `subgroupConcentration` signal.

### Status surface
- Updated the control-plane `Tools` row to show:
  - dominant category
  - dominant subgroup
  - category concentration
  - subgroup concentration
- Kept the tool inventory payload stable for the status and diagnostic surfaces.

## Debt cleanup

- The inventory report now makes the `action` and `admin` buckets legible as subgroups instead of one coarse monolithic bucket.
- `list_tools` and `status-server-details` now share the same canonical inventory source.
- Removed the local registry ownership / ad-hoc helper path that was triggering the watcher.

## Validation

- `list-tools`, `status-system-table`, and `status-summary-payload` tests pass.
- `get_recent_errors()` is clean after the refactor.
- The live server status remains healthy and ready.
