# v0.9.152 - Sprint 15: Import Validation & Relation Path Canonicalization

**Date:** 2026-03-20

## Summary

This release closes the import-validation gap around `#...` package imports, removes the stale pipeline alias from relation persistence, and keeps call-relation writes on the canonical local helper path.

## Key Changes

1.  **Package Import Validation**: `validatePostEditOptimized` now resolves `#...` imports through `package.json#imports` before treating them as valid.
2.  **Watcher Visibility**: validator traces now go through the logger, so broken calls surface in `_recentErrors` instead of only appearing on stdout.
3.  **Canonical Relation Path**: `sqlite-relation-operations.js` now uses the local relation helper instead of importing the pipeline module through a stale alias.
4.  **Call Relation Stability**: `call-relations-linkage.js` keeps the inline persistence path that avoids the broken wrapper / call-chain drift.
