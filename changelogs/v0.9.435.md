# v0.9.435 — Folderize Family Naming Repair

## Bugs Fixed

### 1. Redundant folder names in folderization
- **Before**: `src/shared/compiler/compiler-metrics-snapshot/`
- **After**: `src/shared/compiler/metrics/`
- **Root cause**: `scoreCandidateGroup` used `familyRoot` directly which included directory context prefix
- **Fix**: Strip directory basename prefix from `familyRoot` when building `recommendedFolder`
- **File**: `src/shared/compiler/directory-structure-folderization-analysis.js`

### 2. Redundant file basenames after folderization
- **Before**: `compiler/metrics-snapshot/compiler-metrics-snapshot-helpers.js`
- **After**: `compiler/metrics-snapshot/helpers.js`
- **Root cause**: `buildMoveTarget` used raw basename without stripping family prefix
- **Fix**: Added `normalizeMemberBasename()` to strip `familyRoot-` prefix from file names
- **File**: `src/shared/compiler/directory-structure-folderization-migration.js`

### 3. Missing `history` and `snapshot` suffixes in family detection
- **Impact**: `compiler-metrics-snapshot-history.js` was not grouped with other snapshot files
- **Fix**: Added `'history'` and `'snapshot'` to `FOLDERIZATION_SUFFIXES` set
- **File**: `src/shared/compiler/directory-structure-folderization-analysis.js`

### 4. `buildActualMoveMap` matching wrong files
- **Root cause**: Token-based matching (`snapshot` + `helpers` = score 0.5) matched `compiler-metrics-snapshot-helpers` → `snapshot-summary-helpers`
- **Fix**: Removed `buildActualMoveMap` from folderize flow; plan already knows exact targets
- **File**: `src/layer-c-memory/mcp/tools/folderize-family-import-rewriter.js`

### 5. Orphaned `actualMoveMap` reference
- **Root cause**: After removing `buildActualMoveMap`, `internalRenameMap` still referenced it
- **Fix**: Use `path.resolve(projectPath, target.to)` directly
- **File**: `src/layer-c-memory/mcp/tools/folderize-family-import-rewriter.js`

### 6. `MoveOrchestrator` rewriting imports before folderize rewriter
- **Root cause**: `rewriteMovedFileReferences` changed imports inside moved files BEFORE folderize rewriter had complete picture
- **Fix**: Added `skipSelfRewrite` option to defer import rewriting to folderize rewriter
- **Files**: `src/layer-c-memory/mcp/core/shared/move-orchestrator.js`, `src/layer-c-memory/mcp/tools/folderize-family-plan-runner.js`

### 7. `rewriteIntraFamilyImports` regex not matching `../` imports
- **Root cause**: Regex only matched `./` relative imports, not `../`
- **Fix**: Expanded regex to match both `./` and `../` patterns
- **File**: `src/layer-c-memory/mcp/tools/folderize-family-import-rewriter.js`
