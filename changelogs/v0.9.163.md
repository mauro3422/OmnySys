# v0.9.163 - Sprint 16: Analysis Generation & System Map Recovery

**Date:** 2026-03-21

## Summary

This release formalizes analysis generations as a first-class canonical snapshot and adds a DB-only repair path for persisted system-map support tables. When a failed reanalysis leaves `system_files` or `file_dependencies` empty or stale, the compiler can now rebuild them from the primary SQLite surfaces without falling back to the filesystem.

## Key Changes

1. **Analysis Generations**: the shared compiler now publishes a canonical generation snapshot with drift status, fingerprint, and derived-feature metadata.
2. **Derived Feature Registry**: purpose, archetype, topology, risk, contract, and lifecycle surfaces now live in a canonical registry so downstream policies can extend them consistently.
3. **System Map Recovery**: `system_files` and `file_dependencies` can now be repaired from primary DB data after a locked or partial reanalysis.
4. **DB-Only Export Validation**: export-chain validation and file-query export resolution stay DB-backed while remaining able to resolve re-export aliases and builtin/external modules cleanly.

## Validation

- The new helper modules pass `node --check`.
- The analysis-generation regression test passes.
- The DB repair path was executed successfully and restored the support tables without filesystem fallback.
- The runtime is fresh after cache reload and the live health checks returned `A+`.
