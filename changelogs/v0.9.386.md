# Changelog OmnySys v0.9.386

## Sprint 16: Reconnect Compatibility Hardening and Metrics Trust Plan

### Changed

- Hardened the Streamable HTTP MCP path so degraded JSON-first handshakes from
  Codex/VS Code are normalized before the MCP SDK transport reads them.
- Kept the reconnect fix aligned with the real Node adapter boundary by
  normalizing not only `req.headers`, but also the raw incoming header surfaces
  used by the live HTTP stack.
- Documented the reconnect bug as a transport/session issue with a concrete
  root-cause note, not as a generic daemon failure.
- Added a phased maintenance plan at
  `docs/04-maintenance/system-stability-phased-plan.md` so a lower-level agent
  can continue the work without re-discovering the trust model or the current
  debt priorities.

### Documentation

- Expanded `docs/04-maintenance/daemon-managed-lifecycle-notes.md` with the
  root-cause discovery details for the `406 Not Acceptable` branch that still
  appeared after the first patch.
- Kept the restart policy explicit that `clearCacheOnly` is not enough to prove
  a transport/router fix is live; a real worker reload is required.
