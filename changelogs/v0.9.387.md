# Changelog OmnySys v0.9.387

## Sprint 16: Metrics Dictionary and Cross-Layer Reliability

### Changed

- Added a canonical compiler metric dictionary that explains visible status and
  health metrics in terms of source-of-truth surfaces, SQLite tables, graph
  layers, and caveats.
- Added a cross-layer reliability model that scores runtime, database, graph,
  metadata, and governance trust separately, then rolls them into one global
  reliability score.
- Added a composed `globalHealthScore` so the executive summary no longer
  reflects database health alone.
- Wired the new metric dictionary and reliability fields into compiler metrics
  snapshots, health dashboards, health panels, and compact status summaries.

### Validation

- Targeted Vitest suite passed for:
  - `compiler-metrics-snapshot`
  - `compiler-health-dashboard`
  - `mcp-http-session-routing`
- Shell-driven MCP validation confirmed the live daemon now returns:
  - `globalHealthScore`
  - `reliabilityScore`
  - `metricDictionary`
  - layer-by-layer trust evidence

### Follow-up

- Reduce the new watcher governance debt on the summary layer and dictionary
  helper.
- Continue with the plan phase that maps every major metric to exact SQL
  semantics and removes remaining naming drift between dashboards and tables.
