### **4. Architectural Pattern Detection** (Tarea 4)

**Qué hace:** Detecta patrones arquitectónicos en archivos y sugiere estructura de carpetas para refactorizar.

**Patrones detectados:**
- `GOD_OBJECT` - Archivo con demasiadas responsabilidades
- `ORPHAN_MODULE` - Módulo sin dependientes
- `HELPER_UTILITY` - Helper que debería estar en `/utils/`
- `POLICY_MODULE` - Policy/guard que debería estar en `/compiler/`
- `SERVICE_LAYER` - Service/manager que debería estar en `/services/`

**Features:**
- `detectArchitecturalPattern()` - Detecta God Object, Orphan Module
- `detectHelperUtilityPattern()` - Detecta helpers utilitarios
- `detectPolicyModulePattern()` - Detecta policy modules
- `detectServiceLayerPattern()` - Detecta service layers
- `detectAllArchitecturalPatterns()` - Combina todos los patrones
- `summarizeArchitecturalPatterns()` - Resume en mensaje legible
- `buildGodObjectStructure()` - Sugiere estructura de carpetas

**Archivos:**

### **5. Architectural Debt Score** (Tarea 5)

**Qué hace:** Calcula score de deuda arquitectónica (0-100) basado en 4 categorías ponderadas.

**Fórmula:**
```javascript
score = (
  directoryStructureScore * 0.25 +  // Files en directorios incorrectos
  patternScore * 0.25 +             // God Objects, Orphan Modules
  couplingScore * 0.25 +            // Archivos con >10 imports
  duplicationScore * 0.25           // Duplicados conceptuales
) * 100;
```

**Niveles de severidad:**
- `low`: 0-25
- `moderate`: 26-50
- `high`: 51-75
- `critical`: 76-100

**Output:**
```javascript
{
  overallScore: 35,
  level: "moderate",
  breakdown: {
    directoryStructure: { score: 20, issues: 15 },
    patterns: { score: 45, issues: 7 },
    coupling: { score: 30, issues: 3 },
    duplication: { score: 45, issues: 50 }
  },
  topIssues: [...],
  totalIssues: 75
}
```

**Archivos:**
- `src/shared/compiler/architectural-debt-score.js` (NEW - 401 líneas)
- `src/shared/compiler/index.js` (exports agregados)

**Commits:**
- `162a601` feat: Add Architectural Debt Score (Tarea 5 del Backlog)

## 📊 MÉTRICAS vs HERRAMIENTAS NUEVAS

| Métrica | Fuente | Qué mide | Diferencia con Debt Score |
|---------|--------|----------|---------------------------|
| **Risk Score** | `risk_assessments` DB | Riesgo individual por archivo (1-5) | **Debt Score:** Deuda global (0-100) |
| **Fragility** | `atoms.derived_json` | Fragilidad del código (0-1) | **Debt Score:** Incluye organización + patrones |
| **Coupling** | `atoms.derived_json` | Acoplamiento (0-1) | **Debt Score:** Ponderado con otras categorías |
| **Cohesion** | `atoms.derived_json` | Cohesión (0-1) | **Debt Score:** Score consolidado |

**BÁSICAMENTE:**
- `aggregate_metrics(risk)` → "Estos 46 archivos son críticos"
- `calculateArchitecturalDebtScore()` → "Tu arquitectura tiene 35/100 de deuda, distribuida así..."
