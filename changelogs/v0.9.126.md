# v0.9.126

## MCP Export Chain Validation & Runtime Crash Prevention

### Nueva Herramienta: validate_exports

- creado `validate-exports-chain.js`: Core logic para trazar cadenas de exports
- creado `validate-exports-tool.js`: MCP tool wrapper
- nueva tool `mcp_omnysystem_validate_exports`:
  - Valida que los imports existen en la cadena de exports
  - Detecta errores "module does not provide an export named X" antes del runtime
  - Traza exports transitivos a través de re-exports
  - Usa la DB de OmnySys para validar metadata

**Motivación:** Los crashes por exports rotos solo se detectaban en tiempo de carga. Esta herramienta valida proactivamente las cadenas de exports.

### Nueva API: findAtomByLine

- `atom-query.js`: Nueva función `findAtomByLine(rootPath, filePath, lineNumber)`
- Exportada através de toda la cadena: `atoms/index.js` → `file-query/index.js` → `file-api.js`
- Permite buscar un átomo por número de línea sin acceso directo a DB
- Usada por `context-resolver.js` para seguir patrones canónicos de API

**Fix relacionado:**
- `atom-query.js`: Corregido import de `../core/single-file.js` → `../core/index.js` para usar exports canónicos

### MCP Tools Registradas

- `tool-handler-action.js`: Agregados handlers para:
  - `mcp_omnysystem_safe_edit`: Editor de alto nivel con contexto automático
  - `mcp_omnysystem_get_edit_context`: Obtener contexto de edición para debugging
  - `mcp_omnysystem_suggest_canonical_api`: Detecta acceso directo a DB y sugiere APIs canónicas
  - `mcp_omnysystem_validate_exports`: Valida cadenas de exports

### Compiler: Duplicate Signal Policy Detectors

- `core-policy.js`: Agregadas funciones de decisión de política:
  - `shouldIgnoreConceptualDuplicateFinding()`: Ignora duplicados conceptuales de bajo valor
  - `shouldIgnoreStructuralDuplicateFinding()`: Ignora duplicados estructurales de bajo valor
  - `classifyUtilityHelperDuplicate()`: Clasifica helpers utilitarios y sugiere consolidación
- `detectors/index.js`: Exporta nuevas funciones de política
- `index.js`: Re-exporta API de decisión de política

**Patrones detectados:**
- Validación/guard helpers (isValid, hasProperty, ensureExists)
- Transformación helpers (normalize, format, parse)
- Utilitarios genéricos (get, set, build, create, make)

### Semantic Analyzer: Ignorar Nombres Generados por Test Frameworks

- `semantic-analyzer.js`: Agregada función `isTestFrameworkGeneratedName()`
- Agregados patrones para Vitest/Jest:
  - `/^each_/` - Vitest `.each()` generated names
  - `/_arg\d+$/` - Argument placeholders
  - `/^test_each_/`, `/^describe_each_/`, `/^it_each_/` - Dynamic test patterns
- `computeSemanticFingerprint()`: Retorna `'test_framework:generated:skip:skip'` para nombres generados

**Resultado:** Previene falsos positivos de duplicados conceptuales de tests dinámicos.
