# v0.9.172 - Sprint 16: Canonical Surface Audit Unification

## Summary

- Unified the DB-first status surfaces behind a canonical `surfaceAudit` block so summary and explainability read the same generation, freshness, and coverage data.
- Added metadata extraction coverage reporting for atoms, files, and `system_files` with missing/covered field breakdowns and primary issue reporting.
- Tightened compiler governance and reuse policies so the new audit surfaces are treated as canonical contracts rather than ad hoc helpers.

## Details

- `src/shared/compiler/surface-audit.js` now composes generation, data-gateway, database-health, file import evidence, system map persistence, metadata parity, metadata extraction coverage, semantic granularity, and file universe granularity into one audit payload.
- `src/shared/compiler/compiler-diagnostics-snapshot.js`, `src/layer-c-memory/mcp/tools/status-summary.js`, and `src/layer-c-memory/mcp/tools/status-compiler-explainability.js` now expose the unified audit through the same canonical path.
- `src/shared/compiler/metadata-extraction-coverage.js` and `src/shared/compiler/metadata-extraction-coverage-helpers.js` provide DB-only coverage summaries and field-level missing/covered breakdowns.
- `src/shared/compiler/index.js`, `src/shared/compiler/canonical-reuse-guidance.js`, `src/shared/compiler/metadata-propagation-conformance.js`, `src/shared/compiler/policy-conformance.js`, `src/shared/compiler/compiler-contract-layer.js`, `src/shared/compiler/compiler-contract-layer-helpers.js`, `src/shared/compiler/standardization-report.js`, and `src/shared/compiler/standardization-report/recommendations.js` now recognize the new audit surface as canonical governance metadata.
- Added regression coverage for the new surface audit and metadata extraction summaries.
