# v0.9.181 - Sprint 16: Database Health Auto-Repair for System Map Persistence

## What changed
- Made `getDatabaseHealthSummary()` auto-repair `system_map_persistence` before scoring when the mirrored system file surface is stale or empty.
- Added a regression proving that an empty `system_files` mirror is repopulated from the primary file surface before health is reported.

## Why
- The live status surface was still showing `system_files` as empty after a restart even though the primary file surface was available.
- The health summary now self-heals the supported mirror surface so status reflects the real canonical state instead of a stale projection.

## Validation
- `node --check` passed for the touched database-health files.
- `vitest` passed for the new database-health regression.
- `get_recent_errors()` stayed clean after the edit.
