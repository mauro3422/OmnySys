# Changelog v0.9.85 - Complexity Refactoring Sprint

## [0.9.85] - 2026-03-05

### Added
- **Distributed Builder Pattern**: New specialized builders in `tests/factories/parser-test/builders/`:
  - `function.builder.js`: Specialized handler for function and arrow declarations.
  - `class-generator.builder.js`: specialized handler for class structures.
  - `ts.builder.js`: Specialized handler for TypeScript-specific syntax.
- **Risk Strategy Pattern**: New strategies for `RiskScorer` in `src/layer-b-semantic/inference-engine/strategies/`:
  - `ComplexityStrategy`, `CouplingStrategy`, `ErrorHandlerStrategy`, `TestCoverageStrategy`, `ChurnRateStrategy`, `SideEffectStrategy`, and `RecommendationEngine`.
- **Bulk Operations Handlers**: New specialized handlers in `src/layer-c-memory/storage/repository/adapters/handlers/`:
  - `AtomBulkHandler`, `RelationBulkHandler`, and `EventBulkHandler`.
- **Unit Tests**: New comprehensive test suites:
  - `tests/unit/layer-b-semantic/inference-engine/risk-scorer.test.js`
  - `tests/unit/layer-c-memory/storage/repository/adapters/sqlite-bulk-operations.test.js`

### Refactored
- `tests/factories/parser-test/builders/code.builder.js`: Reduced cyclomatic complexity from **45 to 11** by delegating to specialized builders.
- `src/layer-b-semantic/inference-engine/risk-scorer.js`: Reduced cyclomatic complexity from **31 to 6** by implementing the Strategy pattern.
- `src/layer-c-memory/storage/repository/adapters/sqlite-bulk-operations.js`: Decomposed monolithic bulk save logic into an Orchestrator pattern.
- `src/layer-c-memory/mcp-http-proxy.js`: Improved Windows process management, increased port release delay, and implemented flag propagation (`reanalyze`, `clearCache`).

### Fixed
- **Windows Proxy Cycle**: Fixed race condition in port release during rapid restarts.
- **Global Variable Audit**: Verified that reported "globalVariable" semantic connections were primarily analysis metadata/test artifacts; production code remains encapsulated.

### Performance
- Maintained **~2s** bulk indexing performance while improving modularity and testability of the persistence layer.
- Implemented forced GC after high-memory static analysis phases to prevent OOM before semantic analysis.
