# v0.9.159 - Sprint 16: Cleanup Tool Alignment & Entry-Point Detection Dedupe

**Date:** 2026-03-20

## Summary

This release aligns the MCP zombie cleanup tool with the repo's canonical Windows process pattern and removes duplicated entry-point detection logic in the module-system detectors. The cleanup tool now enumerates node processes through `Get-CimInstance`, while event/job detectors share a common scan helper instead of reimplementing the same module walk.

## Key Changes

1. **Canonical Process Enumeration**: `cleanup_mcp_zombies` now uses `Get-CimInstance Win32_Process` instead of the older WMI path.
2. **Explicit Detection Helper**: zombie matching is isolated in a dedicated predicate to keep the cleanup loop focused.
3. **Shared Entry-Point Scanner**: `findEventHandlers` and `findScheduledJobs` now reuse a common module/atom scan helper.
4. **Simpler Cleanup Flow**: the tool now keeps process discovery, matching, and termination in separate steps with the same public behavior.

## Validation

- `cleanup-processes.js` passes `node --check`.
- The tool still protects OmnySys processes and only targets external MCP zombie processes.
- The cleanup path now matches the Windows process handling style already used elsewhere in the repo.
- Event and scheduled-job detectors now share the same scan primitive, reducing duplicated traversal code.
