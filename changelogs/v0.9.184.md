# v0.9.184 - Sprint 16: Canonical Phase 2 Pending Count Helper

## What changed
- Added a single canonical `getPhase2PendingFiles` helper to the shared compiler runtime metrics module.
- Replaced duplicated inline SQL checks for phase 2 pending files across database health, live-row reconciliation, runtime table health, and the compiler diagnostics snapshot.
- Updated the snapshot test mock to follow the new helper contract.

## Why
- The pending phase 2 count was being computed in multiple places with the same SQL, which made the health and reconciliation surfaces harder to keep in sync.
- Centralizing the helper keeps the count cheap, canonical, and easier to reuse without inventing another abstraction.
- The snapshot and health layers now read the same pending-file signal from one source.

## Validation
- `node --check` passed for the touched compiler and runtime modules.
- `vitest` passed for `compiler-diagnostics-snapshot.test.js` and `database-health.test.js`.
- `get_recent_errors()` returned clean logs, with only runtime restart pending for the updated modules.
