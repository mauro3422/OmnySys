# v0.9.89 - Sprint 9: Daemon Stability & Session Persistence (2026-03-05)

Este sprint se centró en blindar el daemon de OmnySys contra crashes accidentales durante el desarrollo y asegurar que las sesiones de trabajo (como las de Antigravity IDE) no se pierdan cuando el servidor se reinicia por cambios de código.

## 🚀 Key Features

### 1. Persistencia de Sesiones MCP (SQLite)
Se eliminó la dependencia exclusiva de la memoria RAM para las sesiones de MCP.
- **SessionManager**: Nueva infraestructura para el ciclo de vida de sesiones.
- **Recuperación Post-Restart**: El worker ahora detecta headers de sesión conocidos y los restaura desde SQLite si el hash de conexión coincide.
- **Pruning**: Motor de limpieza automática para sesiones obsoletas.

### 2. Escudo de Sintaxis (Syntax Shield)
El Hot-Reload Manager ahora es "consciente" de la validez del código antes de intentar inyectarlo.
- **Pre-validation**: Se ejecuta un subproceso de chequeo sintáctico ligero sobre archivos modificados.
- **Fail-safe**: Si el código es inválido, el hot-reload se cancela preventivamente, protegiendo al daemon principal de excepciones fatales.

### 3. Heurísticas de Salud del Daemon
Ampliamos el motor de reglas de `suggest_refactoring` para detectar anti-patrones de arquitectura de daemons:
- **Mutable Global State**: Identificación de variables de estado que impiden reinicios idempotentes.
- **Async Event Rejection**: Detección de callbacks asíncronos en eventos que carecen de handlers de error superiores.
- **Termination Risk**: Alertas sobre cierres forzados de procesos que impactarían negativamente en la persistencia de datos.

## 🛠️ CLI Updates

- **`omny status`**: Rediseño de la interfaz de consola para incluir:
  - Estado de salud detallado del daemon MCP.
  - Recuento de sesiones activas en tiempo real.
  - Versión y herramientas disponibles reportadas dinámicamente.

## 📊 Métricas de Resiliencia
- **Crashes por SyntaxError**: 100% mitigados en entorno de desarrollo.
- **Persistencia de Sesión al Reinicio**: 100% de éxito en reconexión transparente.
- **Tiempo de Recuperación de Sesión**: < 10ms.
