# v0.9.116

## Summary

This release closes the last semantic surface governance warning by teaching the canonical semantic granularity reader to distinguish true legacy rows from already normalized canonical summary rows.

## What Changed

- Tightened semantic summary normalization in `src/shared/compiler/semantic-surface-granularity.js`.
  - Canonical summary rows with `connection_type` values already normalized to `sharedState` or `eventListeners` are no longer treated as legacy by default.
  - Legacy-only advisories now trigger only when persisted rows use non-canonical buckets or carry fallback provenance from `legacy_semantic_connections`.

## Verified Outcome

- `get_server_status().compilerExplainability.semanticCanonicality.status = stable`
- `get_server_status().compilerExplainability.semanticCanonicality.advisoryCount = 0`
- `get_server_status().compilerExplainability.compilerContractLayer.surfaces[semantic_connections].summary = "Semantic summary and canonical adapter are aligned."`
- `aggregate_metrics(pipeline_health).healthScore = 100`
- `aggregate_metrics(pipeline_health).grade = A`
- `aggregate_metrics(pipeline_health).warnings = []`

## Notes

- `semantic_connections` remains an advisory file-level projection by contract.
- `atom_relations` remains the canonical source of truth for semantic detail.
- The change removes false-positive governance noise; it does not collapse the summary/detail distinction.
