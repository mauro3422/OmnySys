# v0.9.453 - Live Folderization Snapshot Bridge Freshness and Canonical Alignment

**Fecha:** 2026-04-17
**Tipo:** Runtime bridge freshness fix + snapshot canonicalization
**Impacto:** Medio-Alto

---

## Resumen

Esta version corrige la discrepancia entre el snapshot canonico de folderization y la salida viva del bridge MCP. El sistema ahora expone de forma consistente la recomendacion correcta de folderization para `ws-client`, incluso despues de reiniciar el worker y el cliente.

El foco de esta iteracion fue cerrar la brecha entre la logica pura del builder y la ruta viva del daemon, de modo que la snapshot visible no quede pegada a una guidance vieja cuando el runtime vuelve a levantar.

---

## Cambios Principales

### 1. Snapshot de folderization alineado con la verdad canonica

- `get_folderization_snapshot` ahora devuelve en la ruta viva la misma recomendacion que ya producia el builder limpio:
  - `ws-client`
  - `src/core/websocket/client/ws-client`
- Se normalizo la salida del bridge para que el payload visible coincida con la snapshot canonica materializada.

### 2. Bridge MCP y sesiones frescas

- Se reinicio el worker y el bridge para eliminar estado stale en la capa de transporte.
- La salida del shell helper y la salida directa del runtime volvieron a coincidir.
- Se redujo la posibilidad de que una sesion viva siga narrando guidance vieja despues de un restart.

### 3. Observabilidad operativa mantenida

- El arranque del sistema sigue mostrando el resumen bootstrap completo.
- `status` sigue sano con base de datos alineada y runtime fresco.
- La snapshot de folderization queda lista para usarse como referencia operativa, no solo como reporte local.

---

## Resultado Operativo

Estado visible tras la iteracion:

- `status`: sano y fresco
- `databaseHealth`: `healthy 100/A+`
- `get_folderization_snapshot`: recomendacion correcta para `ws-client`
- `bridge`: alineado con el builder canonico
