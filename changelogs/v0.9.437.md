# v0.9.437 — WSL MCP Route Isolation & Guard Crash Hardening

## Resumen

Esta iteracion estabiliza el puente WSL de Codex hacia el daemon MCP de Windows y reduce ruido operativo del watcher al endurecer el guard `compiler-policy-conformance`.

## Cambios Clave

- **Aislamiento de rutas MCP en WSL**: el launcher `scripts/mcp/omnysystem-wsl-bridge.sh` ahora genera una `client_route_id` explicita por proceso para evitar colisiones entre launchers concurrentes de Codex.
- **Sync WSL mas robusto**: `wsl-codex-sync` agrega `OMNYSYS_CLIENT_ROUTE_BASE`, refresca wrappers viejos que no lo tenian y preserva route bases custom.
- **Documentacion de evolucion del bug**: se agrego evidencia de la familia `Transport closed` en WSL y se enlazo desde el bug previo de restart/reconnect.
- **Crash guard hardening**: `compiler-policy-conformance-guard` ya no intenta resolver paths con contexto incompleto y los deferred guards de folderization ahora pasan `rootPath` correctamente.

## Detalles

### WSL / MCP

- `scripts/mcp/omnysystem-wsl-bridge.sh`
  - exporta `OMNYSYS_CLIENT_ID`
  - exporta `OMNYSYS_CLIENT_NAME`
  - construye `OMNYSYS_CLIENT_ROUTE_ID` cuando no viene definido
- `src/cli/utils/mcp-standardizer/wsl-codex-sync-helpers.js`
  - agrega `OMNYSYS_CLIENT_ROUTE_BASE`
  - detecta wrappers WSL antiguos que requieren refresh
- `src/cli/utils/mcp-standardizer/wsl-codex-sync.js`
  - adapta configs wrapper-backed viejas si les falta route metadata

### Watcher / Guards

- `src/layer-c-memory/mcp/tools/folderize-family-plan-runner.js`
  - los deferred guards ahora reciben `rootPath`
- `src/core/file-watcher/guards/compiler-policy-conformance-guard.js`
  - sale de forma segura cuando falta `rootPath` o `filePath`

## Tests

  - preservacion de route base custom
