# OmnySys v0.9.143

**Sprint 15: Call Graph Integrity & Fallback Deprecation**
*Fecha: 2026-03-18*

## Logros Principales

1. **Restauración del Grafo de Llamadas (Call Graph)**: Se corrigió el proyector relacional en `sqlite-relation-operations.js`. El sistema ahora utiliza un *Resolvedor Semántico* que cruza el `imports_json` con `calls_json` para trazar los IDs absolutos (`target_id`) de las dependencias inter-archivos. La vista SQL de `call_graph` ahora genera enlaces precisos a lo largo de todo el proyecto (100% CalledBy Resolution).
2. **Corrección de Integridad del Archivero**: Se modificó el `live-row-utils.js` para detener la eliminación prematura de archivos de barril (zero-atoms) de los índices de estado (`risk_assessments`, `system_files`, y tabla `files`), utilizando `compiler_scanned_files` como la única fuente de verdad autoritativa para el ciclo de vida de archivos.
3. **Deprecación de Heurísticas de Respaldo Ciego en MCP**:
   * Las herramientas como `atomic_edit` calculaban el "Blast Radius" utilizando una función temporal (un fallback llamado `findCallersEfficient`) que usaba parseo in-memory limitado a 1000 átomos ciegos.
   * **Desactivación y Guard Gate**: Se reemplazó este fallback destructivo por consultas SQL estrictas de extremo a extremo que consultan la vista `call_graph`. Ahora las herramientas calculan su impacto sin falsos positivos en base a TODA la red neuronal del código. Si el call_graph no está hidratado correctamente, el sistema aborta de forma segura ("Fail-Fast") levantando un error en lugar de procesar ediciones ciegas.
4. **Parches Críticos Finales**: Se corrigió un error en el guard en línea `compiler-policy-conformance` derivado de imports no definidos (`shouldScanCompilerFile`) que saturaba los FileWatchers.
