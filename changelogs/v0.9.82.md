**Date:** 2026-03-04  
**Type:** Bug Fix + Feature  
**Scope:** extractors, storage, MCP tools, pipeline orchestrator

## Bug Fixes

### 🔧 `hasNetworkCalls` always false — `side-effects.js`
- **Root cause:** Only 4 patterns (fetch, axios, XHR, jQuery) — none cover Node.js
- **Fix:** Expanded to 17 patterns: `https.get/request`, `http.get/request`, `got`, `node-fetch`, `superagent`, `needle`, `undici`, `WebSocket`, `socket.io`, `EventSource`, `ky`, `wretch`
- **Test:** 6/6 ✅

### 🔧 `centralityScore` always 0 — `static-insights.js`
- **Root cause:** `persistGraphMetrics()` was an orphan function — never called from pipeline
- **Fix:** Connected at 2 points: 5s after initial boot + at Phase 2 completion

### 🔧 `atom_events` always empty — `sqlite-bulk-operations.js`
- **Root cause:** `saveMany()` never recorded `created`/`updated` events — only `delete()` did
- **Fix:** Added pre-check (1 query per file for existing IDs) + batch event insert post-save, outside the critical transaction

### 🔧 `detect_performance_hotspots` always returns 0 — `detect-performance-hotspots.js`
- **Root cause:** Only 5/11213 atoms had `performance_json` (extractor requires Phase 2 deep scan). Tool returned 0 because it only scored atoms with `performance_json`
- **Fix:** Fallback scoring from `cyclomatic complexity` column when `performance_json` is null. Covers all atoms. Also improves SQL query to fetch `is_async`, `has_error_handling`, `out_degree` for richer heuristics

## New Features

### ✨ `pipeline_health` aggregationType — `aggregate-metrics.js`
New self-diagnostic tool: `aggregate_metrics({ aggregationType: 'pipeline_health' })`

Detects:
1. **Tables with 0 rows** that should have data (with specific diagnosis per table)
2. **Fields with 0% coverage** in atoms table with probable root cause
3. **Orphan pipeline functions** — exported functions with 0 callers matching pipeline name patterns (persist*, analyze*, compute*, etc.)

This tool would have caught all bugs in this session automatically.
