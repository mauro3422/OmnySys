# v0.9.158 - Sprint 15: Import Validation Error Boundary Hardening

**Date:** 2026-03-20

## Summary

This release hardens the DB-only import validation path with explicit error boundaries around recursive dependency loading. If dependency resolution fails, the tool now returns a structured validation error instead of collapsing into an unhandled runtime exception.

## Key Changes

1.  **Dependency Load Boundary**: recursive circular-dependency loading now wraps `getFileDependencies(...)` with explicit error handling.
2.  **Structured Failure Mode**: circular-dependency resolution failures now surface as `VALIDATION_FAILED` from the public MCP tool.
3.  **Regression Coverage**: a DB-only validation regression test now covers the missing-index case, the DB-backed export chain case, and the dependency loader failure case.

## Validation

- The DB-only regression test suite passes.
- `validate-imports.js` still resolves imports and circular dependencies through canonical DB APIs only.
- The remaining watcher duplicate on `performAction` is contractual and does not indicate a DB or export-chain regression.
