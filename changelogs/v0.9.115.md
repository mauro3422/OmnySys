# v0.9.115

## Summary

This release removes an accidental root-level SQLite artifact from version control and makes the repository explicitly ignore that misplaced database family.

## What Changed

- Removed the tracked root file `.omnysys.db` from the repository state.
- Added explicit ignore rules for root-level SQLite artifacts:
  - `.omnysys.db`
  - `.omnysys.db-wal`
  - `.omnysys.db-shm`

## Why

- OmnySys runtime memory and persistence belong under `.omnysysdata/omnysys.db`.
- A root-level `.omnysys.db` was inconsistent with the project contract and created confusing git noise even though the live system already uses `.omnysysdata`.

## Verified Outcome

- Repository state now matches the intended storage contract.
- Existing runtime/database code continues to point to `.omnysysdata/omnysys.db`.
