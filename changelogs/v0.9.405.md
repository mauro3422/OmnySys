# v0.9.405

## Folderization Snapshot Reuse

1. `technical_debt_report` now reuses the latest DB-backed folderization snapshot when available, so naming debt no longer collapses to zero just because the live compiler snapshot was built without folderization context.
2. `get_server_status` now keeps a short TTL cache for repeated panel reads, reducing redundant recomputation while keeping the live error path untouched.
3. Reexported the status response cache through the canonical compiler barrel and removed a private helper import that was still tripping policy drift on the live `status` tool.
4. Rewired `SemanticQueryTool` to import compiler helpers directly instead of via the oversized compiler barrel, reducing the chance of loader cycles in the heavier query tools.
5. Split the duplicate and society query helpers out of `SemanticQueryTool` so the coordinator stays under the complexity threshold and the warning surface stays smaller.
6. Added coverage for the folderization snapshot loader, technical debt integration, status cache reuse, and the new canonical import paths to lock the behavior in place.
