# OmnySys v0.9.88 - Sprint 8 (Integridad y Detección de Infraestructura)

Esta versión completa el **Sprint 8**, enfocado en corregir puntos ciegos críticos en la detección de infraestructura, resolver avalanchas de falsos positivos en la prevención de duplicados, y mejorar la resiliencia del pipeline de extracción.

## 🚀 Mejoras (Features & Enhancements)
- **Detección de Infraestructura de Red**: Agregados patrones para detectar inicializaciones de `express`, `http.createServer` y `app.listen()` en `side-effects.js`. Permite identificar correctamente dependencias de backend y red en la base de datos (pasando de 0 a > 680 detecciones precisas).
- **Extracción Precisa de Variables**: Modificada la lógica de extracción de código fuente (`atom-extractor.js`) para átomos que no son funciones. Se reemplazó el defectuoso `String.prototype.slice(line, line)` por un sistema robusto basado en el split de líneas `code.split('\n')`.

## 🐛 Correcciones (Bug Fixes)
- **Falsos Positivos de Duplicados (Watcher)**: Se ajustó el query SQL de `duplicate-risk.js` para filtrar explícitamente `dna_json != 'null'`. Esto erradicó una avalancha de >8,700 advertencias falsas de la fase de validación en tiempo real.
- **Funciones Huérfanas de Import**: El `function-linker.js` ahora utiliza `path.resolve` para normalizar las importaciones. Esto reconecta exitosamente nodos cruciales del Call Graph que aparecían desconectados debido a importaciones relativas (ej., `../builders/metadata-builder.js`).
- **Crash del File Watcher (Barrel Export)**: Arreglado un error transitivo en el File Watcher (`_isPatternLogicFile is not a function`) exportando correctamente los métodos de validación semántica al prototipo a través del archivo barril (`lifecycle\index.js`).
