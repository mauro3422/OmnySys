# Release v0.9.112 - Contract Taxonomy for Cross-Domain Governance

**Date:** 2026-03-11
**Scope:** General contract classification for duplicate/governance decisions.

## Executive Summary

This release introduces a first transversal **contract taxonomy layer** so OmnySys stops relying on operator or LLM intuition when deciding whether two atoms are truly comparable. Instead of creating special-case logic for AST, Tree-sitter, games, media, or future software domains, the compiler can now classify surfaces by family, flavor, truth role, and representation level before duplicate governance compares them.

## Major Changes

### 1. General Contract Taxonomy
- Added [contract-taxonomy.js](/c:/Dev/OmnySystem/src/shared/compiler/contract-taxonomy.js) as a shared compiler surface.
- New classification dimensions:
  - `contractFamily`
  - `contractFlavor`
  - `truthRole`
  - `representationLevel`
- Added compatibility evaluation so surfaces with incompatible contracts are not treated as equivalent just because names or fingerprints look similar.

### 2. Duplicate Guard Integration
- Wired the taxonomy into [duplicate-conceptual-core.js](/c:/Dev/OmnySystem/src/core/file-watcher/guards/duplicate-conceptual-core.js).
- Conceptual duplicate comparison now skips peers when:
  - contract family differs
  - flavor differs in a non-generic way
  - truth roles conflict
  - representation levels are not comparable
- This turns the earlier Tree-sitter/Babel concern into a general compiler rule instead of a one-off manual judgment.

### 3. Shared Compiler Barrel Update
- Exported the new taxonomy helpers from [index.js](/c:/Dev/OmnySystem/src/shared/compiler/index.js) so guards and future governance tools can reuse the same policy entrypoints.

## Why This Matters

This is the first step toward a domain-agnostic compiler contract system. The same model can now be extended beyond AST concerns to:
- storage surfaces vs caches
- canonical vs advisory metrics
- runtime APIs vs derived summaries
- future game/media/software domains

Instead of inventing ad-hoc guards per technology, OmnySys can evolve one general compatibility model and let the guards consume that.
