# Release v0.9.87

## Reactive Analysis & Detector Standardization

This release introduces the **Reactive Pattern Refresh** system, allowing OmnySys to re-evaluate technical debt and quality metrics in seconds without a full re-index. We've also completed the standardization of the detector ecosystem.

### 🧠 Reactive Pattern Refresh (Sprint 5)
- **Ultra-fast re-calculation**: New `omnysystem refresh` command re-processes all patterns using existing database atoms in **~1.1s** (vs 20s+ for full indexing).
- **Semantic Invalidation**: The file watcher now automatically triggers a global pattern refresh when detector or rule logic is modified.
- **Improved indexer consistency**: `refreshPatterns` reconstructed the systemMap from SQLite atoms.

### 🎯 Detector Standardization (Sprint 4)
- **Rule Engine Pattern**: Unified all major detectors under a modular rule-based architecture:
    - `SqlPatternsDetector`
    - `CouplingDetector`
    - `HotspotsDetector`
    - `SingletonTracker` (Standardized as a detector)
- **SQL Decomposition**: Refactored the monolithic `SqlAnalyzer` into specialized components:
    - [SqlSyntaxExtractor](file:///C:/Dev/OmnySystem/src/layer-a-static/parser/extractors/sql-syntax-extractor.js)
    - [SqlPurposeClassifier](file:///C:/Dev/OmnySystem/src/layer-a-static/parser/extractors/sql-purpose-classifier.js)
- **Complexity reduction**: `SqlAnalyzer` cyclomatic complexity reduced from **75 to <10**.

### 🛠️ CLI & DX Improvements
- Added `refresh` and `rethink` aliases to the CLI.
- Improved logging for background maintenance tasks.
- Enhanced `PipelineRunner` to support state-based re-runs.

### 🐛 Bug Fixes
- Fixed incorrect destructive import of `enhanceSystemMap` in the indexer.
- Resolved race condition in the file watcher's `processChange` when handling multiple rapid edits to detector rules.
