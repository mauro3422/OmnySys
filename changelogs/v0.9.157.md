# v0.9.157 - Sprint 15: Circular Dependency Split for Import Validation

**Date:** 2026-03-20

## Summary

This release splits the circular dependency traversal in `validate-imports` into smaller helpers so the canonical DB-only flow stays readable and stays under the watcher complexity threshold.

## Key Changes

1.  **Circular Graph Split**: dependency graph building and cycle collection are now separated into focused helpers.
2.  **Complexity Reduction**: `computeCircularDependencies` is now an orchestration wrapper rather than a nested traversal block.
3.  **DB-Only Contract Preserved**: the import validation flow still resolves through canonical DB APIs and keeps the `DB_MISSING` failure mode.

## Validation

- `validate-imports.js`, `validate-imports/filesystem-validation.js`, and `validate-exports-chain.js` passed syntax checks.
- The DB-only regression test continues to pass.
- The watcher should now re-evaluate `computeCircularDependencies` with lower complexity after runtime reload.
