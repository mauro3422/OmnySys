# Release v0.9.111 - Governance Stabilization and Duplicate Cleanup

**Date:** 2026-03-10
**Scope:** Runtime freshness, semantic surface stability, duplicate reduction, and safer canonicalization.

## Executive Summary

This release hardens OmnySys around a practical compiler-governance workflow: stale runtime/tool states are surfaced more clearly, semantic summary surfaces are resynchronized with their canonical sources, unsafe consolidations are blocked, and several duplicate clusters were collapsed into compatibility surfaces or shared Tree-sitter utilities without crossing AST paradigms.

## Major Changes

### 1. Runtime Freshness and MCP Status Fidelity
- Added explicit runtime freshness provenance so `get_server_status()` and `_recentErrors` distinguish between:
  - fresh runtime code
  - pending manual runtime restarts
  - stale tool modules
- Fixed recent-notification/status reporting so restart-required states no longer masquerade as fresh runtime telemetry.
- Improved MCP runtime visibility around stale module conditions and restart lifecycle.

### 2. Semantic Surface Resynchronization
- Reworked semantic summary persistence so `semantic_connections` is regenerated from canonical `atom_relations` rather than drifting as an independent legacy truth.
- Wired incremental analysis/file-watcher paths to resync semantic summary data after `shares_state` persistence.
- Verified `semantic_connections` as healthy advisory surface (`29`) backed by canonical `atom_relations`, eliminating the earlier material drift.

### 3. Technical Debt and Duplicate Signal Quality
- Split conceptual duplicate reporting into raw vs actionable buckets and exposed noise-class metadata in technical debt surfaces.
- Centralized V4 fingerprint/chest/noise classification to reduce lifecycle/test boilerplate pollution in debt reporting.
- Tuned duplicate guards so no-op runs and low-signal cases produce less noisy telemetry.

### 4. Safer Consolidation and Compatibility Surfaces
- Hardened conceptual-cluster consolidation so it no longer emits invalid wrappers/importless delegations.
- Restored public contract compatibility for `json-reader.fileExists`, then redirected it to the canonical filesystem helper instead of keeping duplicate logic.
- Converted multiple stale/legacy duplicate surfaces into explicit compat reexports, including:
  - tunnel vision CSV export
  - module-system impact/system graph wrappers
  - several AST/data-flow helper surfaces

### 5. Tree-sitter-Safe Data Flow Consolidation
- Consolidated duplicated Tree-sitter `getCalleeName` helpers into the existing shared Tree-sitter utility surface:
  - `src/layer-a-static/extractors/data-flow/utils/ts-ast-utils.js`
- Updated output/transformation extractor consumers to use the shared Tree-sitter helper without crossing into Babel AST helper contracts.
- Removed additional local Tree-sitter helper duplication such as `getAssignmentTarget` where safe.

### 6. Stats and Version/Event Persistence Fixes
- Restored local `getStats()` surfaces where `StatsPool` generic keys had broken telemetry expectations.
- Fixed bulk persistence paths so `atom_versions` and `atom_events` are written again.
- Prevented stale/generic stats access from silently degrading system health surfaces.

## Metrics & Impact
- **Semantic summary drift:** resolved; `semantic_connections` now matches its canonical advisory projection.
- **Duplicate groups:** reduced from the post-chest high-water mark down to **51** active logic duplicate groups during verification.
- **Watcher/runtime fidelity:** `_recentErrors` and status now accurately reflect runtime freshness and restart requirements.
- **Crash prevention:** restored missing `fileExists` export contract that had crashed the MCP worker on startup.
