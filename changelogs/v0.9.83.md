# v0.9.83 - System Stabilization & Functional Isomorphism

## Visión General
Esta versión se enfoca en resolver cuellos de botella críticos del pipeline (deadlocks) y en refactorizar el adaptador principal de base de datos para reducir la deuda técnica bruta. Además, introduce el tan esperado motor de análisis semántico profundo para detectar Isomorfismo Funcional.

## 🐛 Fixes Críticos
- **Phase 2 Deadlock Resuelto:** Se corrigió un severo error en `atom-extractor.js` donde los átomos de tipo `sql_query` no estaban siendo marcados como indexados (`isPhase2Complete`). Esto destraba el pipeline y permite que el 100% de los archivos completen su análisis profundo.

## 🛠 Refactoring de Deuda Técnica
- **SQLiteBulkOperations God Object Desmantelado:** 
  - Se redujo el Score de Complejidad Ciclomática del método core `saveMany` de **45 a 14 puntos**.
  - La lógica utilitaria de Hash fue aislada en su propio módulo nativo (`helpers/hashing.js`).
  - Todo el flujo secundario de tracking e historicidad (Fase 4: inyección de `atom_events` y `atom_versions`) fue abstraído y encapsulado, protegiendo el throughput del bulk insert transaccional.

## ✨ Nuevas Capacidades (Layer B - Semantic)
- **Motor de Isomorfismo Funcional (Duplicate Detection 2.0):**
  - Ya no dependemos únicamente de comparaciones brutas de `dna_json` (AST textual).
  - Implementación matemática avanzada usando `GROUP_CONCAT` en SQLite para agrupar funciones con el mismo `is_async`, misma firma de entrada (`parameter_count`), y el **mismo árbol determinístico de dependencias** (llamadas a funciones y objetos idénticos).
  - Integrado transparentemente en el MCP Tool `aggregate_metrics` (vía `aggregationType='isomorphism'`).

## 📊 Estado Actual del Sistema (Health Check)
- Atoms Indexados: `11,300+`
- Semantic Connections de Shared State e Isomorfismo funcionales al 100%.
- El sistema es capaz ahora de diferenciar entre deuda técnica textual (clones copy-paste) y deuda técnica funcional (dos funciones haciendo lo mismo con distinto código).
