# v0.9.95 - Compiler Canonicalization & Runtime Self-Healing

**Fecha:** 2026-03-06  
**Tipo:** Architecture + Runtime Hardening  
**Impacto:** Alto

---

## Resumen

Esta versión consolida OmnySys como un compilador vivo para agentes. La prioridad de esta tanda fue convertir heurísticas dispersas en APIs canónicas reutilizables, endurecer el runtime MCP ante hot-reloads, y volver honesta la telemetría del compilador.

Watcher, MCP tools, health checks y capas de análisis ahora comparten mucha más política común en vez de reimplementar SQL, hashes, escaneos de archivos o reglas de remediación por separado.

---

## Runtime MCP

- Arranque estandarizado en `stdio -> bridge -> proxy -> worker`.
- Deduplicación de sesiones endurecida con cleanup de arranque, reservas in-flight por `client_id`, y persistencia más consistente de filas activas/inactivas.
- `reindexOnly` preserva el proceso, refresca metadata de cache y relanza Phase 2.
- El bridge falla rápido con errores MCP `retryable` durante reinicios del daemon.
- El bridge auto-recupera mejor sesiones stale/no-initialized.
- Proxy y bridge comparten locks de ownership para evitar proxies duplicados durante hot-reload.

Familias funcionales consolidadas:

Familias arquitectónicas y de buenas prácticas agregadas:

También se agregó `standardization-report` para mostrar familias ya canonizadas, gaps de adopción y próximos frentes de consolidación.

## Watcher & Telemetry

- El FileWatcher volvió a drenar correctamente su cola de compilación en vivo.
- Se restauró la observabilidad útil en terminal: cambios detectados, resúmenes por archivo, persistencia/clear de issues y notificaciones de guards.
- `_recentErrors` fusiona logger transitorio con alerts persistidas del watcher.
- Las alerts del watcher ahora usan payload, lifecycle y reconciliación canónicos.
- `health` y `pipeline_health` distinguen “dato faltante” de “cero real”.
- `status` y `/health.background` exponen mejor `compilerReadiness`, progreso de Phase 2 y fallback `fast_phase2`.

## Semantic Persistence & Physics

- Se corrigió la persistencia de `dna`, `data_flow` y `signature` hacia SQLite.
- Los converters ya no descartan scores derivados (`fragility`, `cohesion`, `testability`, `importance`).
- `persistGraphMetrics()` backfillea mejor físicas y métricas de grafo.
- La detección de duplicados dejó de tratar `dna_json = 'null'` como DNA válida.
- Se centralizó la política de duplicados y el watcher ahora puede detectar mejor clones renombrados con huella estructural.
- Dead code y pipeline orphans ahora tienen clasificación, reporting y remediación canónica.

## Estado

Lo que mejora:
- Menos deriva entre watcher, tools MCP y análisis compartido.
- Menos SQL/heurística duplicada en capas distintas.
- Mejor resiliencia del daemon durante hot-reload.
- Telemetría más honesta sobre deuda real del compilador.

Lo que sigue abierto:
- Adopción incompleta de algunas familias canónicas (`service_boundary`, `canonical_extension`, `async_error`, `testability`, `semantic_purity`).
- Complejidad alta en algunos módulos del propio compilador/runtime.
- Algunos clientes MCP todavía pueden conservar sesiones stale después de reinicios reales del worker.
