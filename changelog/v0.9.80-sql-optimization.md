# v0.9.80 - SQL Optimization & Schema Drift Extinction (Phase 2)

**Date:** 2026-03-03

## Overview
This release finalizes the second phase of the OmnySys 2026 Optimization Track, focusing on Technical Purity and SQLite Database Optimization. It eradicates Schema Drift false positives and drastically reduces N+1 Query patterns by leveraging advanced relational Window Functions.

## 🚀 Key Improvements

### 1. Extinción del Schema Drift (AST Refactoring)
* **Contexto:** El analizador de salud reportaba +17 errores de "schema drift" en los SQL\_QUERYs debido a variables mal interpretadas del AST y esquemas desactualizados.
* **Fix A:** El `tree-sitter-sql` parser walker (`sql.js`) ahora ignora el placeholder interno `_tbl_`, evitando que se confunda con una columna real SQLite de la base de datos tras ofuscar un Template Literal `\${var}`.
* **Fix B:** Modificamos el `omnysys-health-detector.js` para extraer la verdad absoluta estructurada importando dinámicamente el `schema-registry.js`, alineando el verificador con la DB real.
* **Impacto:** Los falsos positivos de Schema Drift bajaron de 17 a 0.

### 2. Eliminación de N+1 Queries (JOIN Candidates & Window Functions)
* **Contexto:** Las herramientas de semántica (Event Patterns, Race Conditions, Async Analysis) requerían recuperar el `COUNT(*)` total (para la paginación de metadata) separadamente de la propia búsqueda indexada con `LIMIT OFFSET`.
* **Fix:** En el MCP tool `semantic-query-tool.js` modificamos las funciones masivas de búsqueda transaccional para inyectar sentencias Window: `SELECT COUNT(*) OVER() as total_count ...`. 
* **Impacto:** Se eliminaron las consultas de pre-conteo separadas, unificando el tráfico a la SQLite y abatiendo drásticamente el detector de `join-candidates`.

### 3. Unified \`fix-imports\` Fallback Strategy
* **Contexto:** La tool MCP `fix-imports` buscaba resoluciones usando un `try/else` bloque en el JS sobre dos consultas independientes (`atoms` vs `files`) causando ineficiencia funcional. 
* **Fix:** Combinadas en un único constructor `UNION ALL` transaccional.
