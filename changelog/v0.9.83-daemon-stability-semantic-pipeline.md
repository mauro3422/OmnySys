# v0.9.83 тАФ Daemon Stability & Semantic Pipeline Fix

**Date:** 2026-03-04  
**Type:** Bug Fixes + Stability  
**Status:** тЬЕ Released

---

## Summary

This version fixes a cascade of 7 bugs that were preventing: the MCP daemon from surviving restarts, semantic connections from ever being persisted to SQLite, and multiple MCP tools from returning correct data.

---

## Bugs Fixed

### ЁЯЫбя╕П Bug 1 тАФ Daemon kills itself on `restart_server`
**Root cause:** The component restart path always ran `LayerAAnalysisStep` (~10-30s). During that time `server.initialized = false`, causing VS Code to detect the task as "completed" and re-launch it. The new process found port 9999 busy and exited, breaking the MCP connection permanently.

**Fix:** `restart-server.js` тАФ default restart now only restarts the Orchestrator (<1s). Layer A only runs when `reanalyze: true` is explicitly passed. The MCP HTTP server stays alive throughout.

**New restart modes:**
| Option | Time | Effect |
|--------|------|--------|
| `clearCacheOnly: true` | <100ms | Flush RAM cache + tool registry |
| `clearCache: true` (default) | <1s | Cache + Orchestrator restart |
| `reindexOnly: true` | ~10s | Force Layer A (DB preserved) |
| `reanalyze: true` | ~30s | Wipe DB + full Layer A reindex |

---

### ЁЯФС Bug 2 тАФ `aggregate_metrics: risk` returned random keys
**Root cause:** `risk-handler.js` used `_category_${category}_${Math.random()}` as fallback `file_path` when a risk item had no real path. This created garbage keys like `_category_scores_0.502...` in `risk_assessments`.

**Fix:** Skip items without a real `file_path` entirely тАФ they're metadata blobs, not file risks.

---

### ЁЯУК Bug 3 тАФ `status.metadata.totalFunctions` desync
**Root cause:** `status.js` read from in-memory metadata cache which could be stale.

**Fix:** Added `liveAtomCount` and `liveFileCount` via direct `SELECT COUNT(*)` from SQLite atoms table.

---

### ЁЯФм Bug 4 тАФ `health` returned 0/0 for all physics vectors
**Root cause:** `aggregate-metrics.js` health case returned hardcoded zeros.

**Fix:** Direct SQLite query: `SELECT COUNT(*), AVG(fragility_score), AVG(coupling_score), AVG(cohesion_score), AVG(importance_score), AVG(centrality_score) FROM atoms`. Now returns real values like `avgCoupling: 0.131, avgCohesion: 0.397`.

---

### ЁЯФМ Bug 5 тАФ HotReloadManager crashes on every file save
**Root cause:** `ReloadHandler._executeStrategy(filename, moduleInfo)` called `moduleInfo.type` without null guard. `ModuleClassifier.classify()` returns `null` when a file doesn't match any reloadable pattern тАФ crashing with `Cannot read properties of undefined (reading 'name')`.

**Fix:** `reload-handler.js` тАФ null guard: if `moduleInfo === null`, skip silently. Also: `HotReloadManager.index.js` already had this guard тАФ the crash was in a secondary code path.

---

### ЁЯТ╛ Bug 6 тАФ `aggregate_metrics: patterns` always returned 0 semantic connections
**Root cause (chain of 5 issues):**

1. `system-map-enhancer.js` stored 264 connections in `enhanced.connections.{sharedState, eventListeners, routes...}` (structured object) but `persistSystemMapToDb` read `systemMap.semanticConnections` (flat array). Key mismatch = nothing ever reached SQLite.

2. `saveSemanticData` was declared `async` but called synchronously inside a `db.transaction()` callback (better-sqlite3 transactions are fully synchronous). The function body was scheduled as a microtask AFTER the transaction committed тАФ the inserts ran outside the transaction.

3. `semantic_issues` INSERT had `NOT NULL constraint failed: semantic_issues.file_path` тАФ some issues had `undefined` filePath. This caused the **entire transaction to rollback**, silently wiping `semantic_connections` too.

4. `SQLiteAdapterCore.saveSystemMap()` was synchronous but called `async persistSystemMapToDb` without awaiting it тАФ unhandled rejection when the transaction threw.

5. `aggregate-metrics.js` patterns case queried with `this.repo?.db` тАФ but `this.repo` was never `null` (GraphQueryTool.execute initializes it), however the block logic was using `getRepository(this.projectPath).db` which was the correct fix.

**Fixes applied:**
- `system-map-enhancer.js` тАФ adds `enhanced.semanticConnections` flat array alongside the structured `.connections` object
- `system-map.js` тАФ `persistSystemMapToDb` normalizes both shapes (flat array OR structured `.connections.*` object)
- `semantic-handler.js` тАФ removed `async` from `saveSemanticData`, `loadSemanticConnections`, `loadSemanticIssues` (all calls are sync better-sqlite3)
- `semantic-handler.js` тАФ filters `semantic_issues` to only entries with a valid `filePath` before INSERT
- `sqlite-adapter-core.js` тАФ `saveSystemMap` is now `async` and properly `await`s `persistSystemMapToDb`
- `save.js` тАФ `saveEnhancedSystemMap` now `await`s `repo.saveSystemMap()`
- `aggregate-metrics.js` тАФ patterns case uses `this.repo?.db` (set by GraphQueryTool.execute)

**Result verified live:** `semanticConnections.total: 264` with breakdown by type.

---

### ЁЯЫая╕П Bug 7 тАФ `restart_server` schema missing new options
**Fix:** `tools/index.js` тАФ added `clearCacheOnly` and `reindexOnly` to the MCP tool inputSchema so agents can discover and use them correctly.
