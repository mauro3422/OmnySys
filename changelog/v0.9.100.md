# v0.9.100 - Centrality Coverage Adoption

Date: 2026-03-06

## Summary

Centrality-coverage reporting was promoted to a canonical shared/compiler surface and wired into the health and pipeline status flows, removing the last ad hoc read-path for that policy family.

## Highlights

- Added `summarizeFieldCoverageRow` and `summarizeCentralityCoverageRow` as canonical helpers in `src/shared/compiler/signal-coverage.js`.
- Exported the new helpers through `src/shared/compiler/index.js`.
- Updated `health-handler.js` and `pipeline-health-handler.js` to consume the canonical coverage surfaces instead of open-coding coverage summaries.
- Aligned compiler standardization/explainability so `centrality_coverage_adoption` no longer appears as a missing canonical API once the adoption is present.

## Result

- Centrality-coverage adoption became visible and explicit in the compiler explainability layer.
- MCP health/status consumers now read the same centrality-coverage contract.
